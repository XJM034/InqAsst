# 全部页面静态数据转真实清单
日期：2026-04-18

## 返工说明

这份清单已按“前端 -> 后端接口 -> service -> repository -> 数据库表”重新核验。

本次返工后的判定规则：

1. 不能只因为前端在调用接口，就判定“真实化完成”。
2. 必须能继续追到后端 controller / service / repository。
3. 必须继续追到真实表结构；若只有内存态、mock、浏览器态或临时壳，则不能算完成。
4. 页面状态只使用：
   - `已完成`：前端、后端、数据库链路都已核到，且运行时主链路不再读 mock/draft。
   - `处理中`：前端已部分改造，但后端或数据库支撑仍有缺口，或仍保留 legacy 包装路由。
   - `暂缓`：业务方案未定、当前明确不开发，或当前无静态数据问题且不作为阶段阻塞项。
   - `未开始`：尚未完成这轮全链路核验。

当前范围备注：学生的批量导入功能因业务方案尚未设计完成，当前先不开发，只保留页面盘点、缺口记录和风险说明，不纳入本轮真实化开发与联动改造范围。
补充备注：如果某个页面已经没有静态数据、Mock、假链路问题，即使仍保留 legacy 路由或兼容壳，也可以先不处理，不作为当前阶段阻塞项。
