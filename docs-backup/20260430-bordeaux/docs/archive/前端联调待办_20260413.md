## 3. 学生新增 / 编辑页可以接真接口了

### 3.1 新接口

读取详情：

- `GET /api/admin/courses/{courseId}/students/{studentId}`

按校区实时搜索已有学生：

- `GET /api/admin/campuses/{campusId}/students/search?q=&limit=10`

把已有学生加入课程名单：

- `POST /api/admin/courses/{courseId}/students/existing`

临时新增并加入课程名单：

- `POST /api/admin/courses/{courseId}/students`

编辑：

- `PUT /api/admin/courses/{courseId}/students/{studentId}`

### 3.2 请求体

已有学生入课：

```json
{
  "studentId": 123
}
```

临时新增学生：

```json
{
  "name": "王一诺",
  "homeroomClassId": 10
}
```

字段含义：

- `studentId`
  - 数据库学生主键
  - 只用于“已有学生入课”请求
- `name`
  - 学生姓名
- `homeroomClassId`
  - 行政班主键

补充约定：

- 临时新增学生时，前端不再生成 `TEMP-*` 学生 ID
- `externalStudentId` 当前不再作为页面必填输入
- 临时学生保存成功后，由后端返回数据库 `studentId`

### 3.3 响应体 data

```json
{
  "studentId": 123,
  "externalStudentId": "STU-05231",
  "studentName": "王一诺",
  "homeroomClassId": 10,
  "homeroomClassName": "五年级1班",
  "enrolledInCourse": true
}
```

当前页面展示约定：

- 管理员看到的“学生ID”统一显示 `studentId`
- `externalStudentId` 不再以“学生ID”名义展示

### 3.4 前端动作

- `app/admin/course-settings/[courseId]/students/new/page.tsx`
  - 不再首屏全量拉 `/api/admin/students?page=&size=`
  - 输入至少 2 个字后，调 `GET /api/admin/campuses/{campusId}/students/search?q=&limit=10`
  - 选择已有学生后，保存调 `POST /api/admin/courses/{courseId}/students/existing`
  - 未命中已有学生时，保存调 `POST /api/admin/courses/{courseId}/students`
- `app/admin/course-settings/[courseId]/students/[studentId]/edit/page.tsx`
  - 首屏先调 `GET /api/admin/courses/{courseId}/students/{studentId}`
  - 保存时调 `PUT /api/admin/courses/{courseId}/students/{studentId}`
- 表单里“学生ID”字段：
  - 只读显示数据库 `studentId`
  - 不再要求管理员填写 `externalStudentId`

### 3.5 行政班数据来源

继续用现有接口：

- `GET /api/admin/campuses/{campusId}/classes`

前端动作：

- 新增 / 编辑页不要再用纯文本输入行政班
- 改成下拉选择
- 页面要先拿当前管理员校区 `campusId`

## 4. 临时代课老师必须切到“课节维度”

后端业务口径已经按你的确认实现：

- 不是永久改课程老师
- 是“当前课节 / 当天临时点名老师”

这意味着前端不能再只拿 `courseId` 做老师设置页主键，必须带 `courseSessionId`。

### 4.1 新接口

读取当前课节老师设置：

- `GET /api/admin/course-sessions/{courseSessionId}/roll-call-teacher`

读取可选老师：

- `GET /api/admin/course-sessions/{courseSessionId}/roll-call-teacher/options`

设置临时老师：

- `PUT /api/admin/course-sessions/{courseSessionId}/roll-call-teacher`

恢复默认老师：

- `DELETE /api/admin/course-sessions/{courseSessionId}/roll-call-teacher`

创建外部老师并直接设为临时老师：

- `POST /api/admin/course-sessions/{courseSessionId}/roll-call-teacher/external`

### 4.2 `GET /roll-call-teacher` 返回重点字段

- `defaultTeacherId`
- `defaultTeacherName`
- `currentTeacherId`
- `currentTeacherName`
- `currentTeacherPhone`
- `currentTeacherCampusId`
- `currentTeacherExternal`
- `temporaryTeacherAssigned`

### 4.3 `GET /roll-call-teacher/options` 返回重点字段

- `teacherId`
- `teacherName`
- `phone`
- `campusId`
- `externalTeacher`
- `selected`
- `defaultTeacher`

### 4.4 前端动作

- `app/admin/emergency/page.tsx`
  - 入口链接不要只带 `courseId`
  - 必须把 `sessionId` 一起带下去
- `app/admin/emergency/course/[courseId]/page.tsx`
  - 建议改成课节维度页面，至少要能拿到 `courseSessionId`
  - 当前页显示的“当前点名老师 / 默认老师”改为读取新接口
- `app/admin/emergency/course/[courseId]/select-teacher/page.tsx`
  - 读取老师列表改成调 `.../roll-call-teacher/options`
  - 保存改成 `PUT .../roll-call-teacher`
- `app/admin/emergency/course/[courseId]/external-teacher/page.tsx`
  - 保存改成 `POST .../roll-call-teacher/external`
- “恢复默认老师”按钮改成调 `DELETE .../roll-call-teacher`

### 4.5 重要说明

- 课程详情接口里的 `teachers`
  - 仍然是课程级老师绑定
  - 不能再当“当前课节点名老师”来源

也就是说，老师设置相关页面以后不要再依赖：

- `GET /api/courses/{courseId}`

来判断当前临时代课老师。

## 6. 外部老师口径

后端已按当前业务实现：

- 外部老师允许 `campusId = null`
- 外部老师不进入“本校教师列表”主接口
  - `GET /api/admin/teachers` 仍返回本校老师
- 但如果某个课节当前已经绑定外部老师，
  - `GET /roll-call-teacher`
  - `GET /roll-call-teacher/options`
  会把这个外部老师带出来

前端动作：

- 外部老师不要塞进普通教师管理页
- 只在课节级临时代课页面里处理
