# 设计思路

> **v1.8 更新 (2026-03-23)**：根据甲方最新反馈更新。本期回到双角色架构（老师 + 管理员），v2 的助教端、通知系统、推送系统全部延后。详见 `设计文档_v1.8_开发指导.md`。

---

## 1. 全局设计规范 (Global Design Principles)

- **核心理念**：极简信息架构，高容错率，上帝视角。
- **角色架构**：本期只做双角色（老师端 + 管理端）。
- **校区权限**：管理员单校区绑定不可切换；老师可跨校区，按天自动显示当天所在校区信息。
- **地点格式标准**：统一采用层级化文本 `楼栋 > 楼层 > 教室号`（例如：**综合楼 A 栋 > 3 层 > 3年级 2 班**），字号需醒目。
- **视觉分类系统**：采用五色标签/图标系统对应五大课程类别，建立视觉条件反射：
    - **科创** (蓝色系) / **人文** (青色系) / **体育** (橙色系) / **劳作** (绿色系) / **艺术** (紫色系)。
- **标签系统**：删除学生个人特征标签（如"调皮""敏感"等），保留课程类别标签。

## 2. 界面布局建议：用户端 A（外部上课老师）

**定位**：智能行程单 (Smart Itinerary)。

**目标**：老师无需搜索，系统根据账号直接告知“何时、去哪、哪个校区”。

### 页面结构：首页 (Teacher_Home)

### A. 顶部导航区 (Top Nav)

1. **用户信息**：左侧显示头像及欢迎语（如：“李老师，下午好”）。
2. **周历切换器 (Weekly Calendar)**：
    - 横向排列 `周一` 至 `周五`。
    - **状态标记**：默认选中“今天”；有排课的日期下方显示蓝色圆点标记。
    - **交互**：点击不同日期，下方的任务卡片内容随之刷新。

### B. 核心任务卡片 (Mission Card)

*若当日有课，显示以下卡片（支持多张堆叠）：*

1. **卡片眉部（关键信息）**：
    - **校区标识**：通栏深色标签，醒目显示 **📍 [校区名称]**（如：成华校区）。防止老师跑错校区。
2. **卡片主体（课程信息）**：
    - **背景色**：根据课程类别显示淡淡的主题色。
    - **第一行**：
        - **时间**：`16:30 - 17:30`（特大字号，加粗）。
        - **课程**：**无人机飞行 A 班** + 类别图标（✈️）。
    - **第二行（视觉焦点 - 地点）**：
        - 独立色块容器，显示层级文本：**科技楼 > 2 层 > 205 教室**。
        - 右侧预留 **[导航]** 图标按钮。
3. **卡片底部（操作区）**：
    - ~~按钮 1：**查看班级名单**~~ （v1.8 删除，本期不提供学生名单）。
    - 按钮 2：**联系管理老师**（一键拨号，用于突发情况）。

### C. 空状态 (Empty State)

- 若选中日期无课，显示插画及文案：“该日无教学安排”。

## 3. 界面布局建议：用户端 B（校区管理老师）

**定位**：全能搜索与管控台 (Omni-Search Console)。

**目标**：基于“上帝视角”，在信息碎片化的情况下，快速定位**学生**、**老师**或**课程**。

### 页面结构：首页 (Admin_Home)

### A. 超级搜索框与混合预览 (Omni-Search Input)

*这是页面的核心入口，支持对“人”和“课”的全局模糊检索。*

1. **搜索框 (Input)**
    - **Placeholder**：`搜学生、老师姓名、课程...`
    - **交互逻辑**：输入过程中触发**即时联想 (Type-ahead)**。
    - **搜索方式**：支持汉字、拼音全拼、拼音首字母、**同音字匹配**（如输入"李嘉树"能匹配到"李佳澍"）。
    - **结果排序**：默认优先显示学生结果（大多数场景是搜索学生）。
2. **混合预览列表 (Polymorphic Preview)**
    - *触发条件*：输入字符（拼音/汉字）后，键盘上方弹出半屏列表。
    - *展示逻辑*：同时匹配学生库、教师库和课程库，并在列表中标记身份。
    - *列表项样式示例*：
        - `[学生]` **李景行** —— 5年级 3 班
        - `[学生]` **李京星** —— 4年级 8 班
        - `[老师]` **王力** —— 无人机飞行课程
        - `[课程]` **乐高机器人 A 班** —— 16:30 / 科技楼 205
    - *交互*：点击某一项，进入对应的结果详情卡片。

### B. 级联筛选器 (Cascading Filter Funnel)

*位于搜索框下方，辅助缩小“找学生”的范围。*

1. **第一排：课程类别**（优先判断装备）：
    - `全部` `体育` `科创` `艺术` `劳作` `人文`
2. **第二排：年级**（其次判断身高）：
    - `一年级` `二年级` `三年级` ... `六年级`
3. **第三排：行政班级**（最后询问 - **动态显示**）：
    - *默认状态*：隐藏或置灰。
    - *激活逻辑*：当用户在第二排选中某个年级（如“三年级”）后，第三排点亮并显示该年级下的班级（`3.1班` `3.2班`...）。

### C. 结果详情卡片 (Result Cards)

*根据选中对象的类型（学生/老师/课程），动态渲染三种不同的卡片结构。*

**布局一：当目标是 [学生]**

- **左侧：生物特征**
    - **学生照片**（大图，点击可放大）。
    - 若无照片，显示性别图标。
- **右侧上部：身份确权 (Verification)**
    - **姓名**：**李景行**
    - **行政班级**：**5 年级 3 班**
    - **校验信息**（灰色小字，用于管理员口头核对）：
        - `班主任：王某某` （话术：你班主任姓什么？）
        - ~~`生日：08-12`~~ （v1.8 删除，不再需要生日校验）
- **右侧下部：核心指引**
    - **课程**：**乐高机器人** (带颜色标签)
    - **地点**：<span style="color:red; font-weight:bold; font-size:large">科技楼 A 栋 > 2 层 > 205</span>

**布局二：当目标是 [老师]**

- **左侧：身份特征**
    - **教师照片**（或通用教师图标）。
- **右侧上部：身份与联系**
    - **姓名**：**王力**
    - **身份**：`某某机构签约教师`
    - **操作**：**电话图标**（点击直接拨号）。
- **右侧下部：当前任务**
    - **状态**：`正在上课` 或 `即将开始`
    - **课程**：**无人机飞行 A 班**
    - **地点**：<span style="color:red; font-weight:bold; font-size:large">科技楼 A 栋 > 2 层 > 205</span>

**布局三：当目标是 [课程]**

- **左侧：类别特征**
    - **类别图标**（如：齿轮/画板/足球）。
    - 背景为该类别的标准色。
- **右侧上部：课程概况**
    - **课程名称**：**乐高机器人 A 班**
    - **时间段**：`16:30 - 17:30`
    - **班级人数**：<span style="text-decoration: underline; color: blue;">应到 25 人</span> (点击弹出该班所有学生名单，用于点名/核对)。
- **右侧下部：核心指引与管控**
    - **授课教师**：**王力** (带电话图标)。
    - **地点**：<span style="color:red; font-weight:bold; font-size:large">科技楼 A 栋 > 2 层 > 205</span>

## 4. 交互流程场景 (User Flow Scenarios)

### 场景一：管理员帮助迷路学生（学生模糊查找）

1. **线索**：学生口齿不清说叫 "li jing xing"，看起来个子挺高。
2. **操作**：管理员输入 `ljx`。
3. **初筛**：预览列表出现 `李景行(5年级)` 和 `李京星(1年级)`。
4. **判断**：管理员根据身高，点击 `李景行(5年级)`。
5. **校验**：卡片弹出。管理员看照片确认长相，并问：“你班主任姓什么？” 学生答：“姓王”。—— **身份确认**。
6. **结果**：管理员指引：“去科技楼 205 教室”。

### 场景二：管理员帮助未带工牌的老师（老师查找）

1. **线索**：遇到一位陌生老师，自称姓王，来上“飞机课”。
2. **操作**：管理员输入 `王`。
3. **初筛**：预览列表出现多位姓王的学生和老师。管理员点击 `[老师] 王力 - 无人机课程`。
4. **结果**：卡片显示该老师需前往 **科技楼 205**。
5. **扩展**：如果老师迟迟未到，管理员可直接点击卡片上的**电话图标**进行催促。

### 场景三：管理员巡课或家长询问（课程查找）

1. **线索**：家长问“今天的足球课在哪上？”或者管理员想去足球课巡查。
2. **操作**：管理员输入 `足球`。
3. **初筛**：预览列表显示 `[课程] 足球 U8 班` 和 `[课程] 足球 U10 班`。
4. **选择**：点击 `U8 班`。
5. **结果**：卡片显示 **北操场 3 号区**，以及 **张教练** 的电话。管理员可直接点击人数链接查看学生名单是否到齐。

### 场景四：外部老师自查行程（行程规划）

1. **操作**：老师周一晚上在家打开 App。
2. **查看**：点击顶部周历的“周二”。
3. **结果**：卡片显示 **明天 (周二) - 锦江校区 - 16:30**。
4. **决策**：老师明确了明天要去的校区和时间，提前安排出行。
