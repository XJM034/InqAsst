# 接口与 Mock 冲突：`/teacher/attendance/no-class`

## 1. 冲突等级

`中低`

这页不是 mock 中的核心正式页面，它的冲突主要在于“无课状态”被新增成独立真实路由后，是否与其他教师端空态保持一致。

## 2. mock 依赖的对象

mock 的无课状态更接近一种轻量空态对象：

- 今天没有可点名课程
- 这是正常状态

## 3. 当前接口实际使用的对象

当前页没有复杂数据对象，主要依赖入口分发逻辑判断：

- 今天是否存在真实可点名 session

## 4. 具体冲突点

### 4.1 链路冲突

无课状态不再只是一个空态，而是点名入口分发失败后的正式落点。

### 4.2 统一性冲突

当前同时存在：

- `/teacher/attendance/no-class`
- `/teacher/no-class`

如果两边判断来源不一致，就会出现两套无课语义。

## 5. 处理建议

- 这页的显示依据必须统一来自真实课程/session 判断。
- 它和 `/teacher/no-class` 需要共享同一套真实无课判定，不允许各自写一套规则。
