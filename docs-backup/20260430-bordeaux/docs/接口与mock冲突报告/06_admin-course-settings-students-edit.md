# 接口与 Mock 冲突：`/admin/course-settings/[courseId]/students/[studentId]/edit`

## 1. 冲突等级

`中`

当前编辑页的数据能力更强，但 mock 的课程名册上下文语义没有被一起保留下来。

## 2. mock 依赖的对象

mock 编辑页依赖的是：

- 当前课程上下文
- 当前学生信息
- 该学生属于当前课程的说明

本质是“课程内学生编辑对象”。

## 3. 当前接口实际使用的对象

当前链路是：

- `fetchCourseDetail`
- `fetchAdminCampusClasses`
- `fetchAdminCourseStudentDetail`

它已经是一个完整的学生实体编辑对象。

## 4. 具体冲突点

### 4.1 对象冲突

mock 是课程链路子对象。

当前是学生实体编辑对象。

### 4.2 字段冲突

当前接口当然给了更多真实字段，但缺少 mock 更需要的：

- `courseContext`
- 课程内归属文案
- 编辑完成后的链路返回语义

### 4.3 链路冲突

从课程名册点进来后，当前页不像在编辑“这门课里的学生”，而更像在编辑“一个学生实体”。

## 5. 处理建议

- 保留真实编辑能力。
- 额外补一个课程视角的包装对象，不要直接把实体编辑对象裸渲染成页面。
- 返回路径和标题都要继续绑定当前课程。
