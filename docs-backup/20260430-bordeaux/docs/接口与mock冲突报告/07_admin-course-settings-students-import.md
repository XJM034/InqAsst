# 接口与 Mock 冲突：`/admin/course-settings/[courseId]/students/import`

## 1. 冲突等级

`中低`

这页的冲突不是“真实接口把 mock 带偏”，反而是“页面看起来已接接口，但底层仍在依赖 mock 形态的数据对象”。

## 2. mock 依赖的对象

mock 导入页需要的是：

- 支持字段说明
- 模板下载信息
- 导入入口说明

这本来就是轻量对象。

## 3. 当前接口实际使用的对象

当前 `getAdminCourseStudentImport` 仍主要返回 mock 形态结构，而不是真实导入结果对象。

## 4. 具体冲突点

### 4.1 对象缺失

当前并没有完整真实的“导入任务对象”，例如：

- 模板来源
- 上传结果
- 失败明细
- 导入统计

### 4.2 动作冲突

mock 的动作只是演示“可导入”。

当前页面如果要纳入真实业务，就需要完整导入链路对象。

## 5. 处理建议

- 这页 UI 可先保持，但数据层必须尽快替换成真实导入对象。
- 重点是补真实导入接口对象，避免页面停留在“真实页面壳 + mock 形态数据”状态。
- 这页是必须清理 mock 依赖的页面之一。
