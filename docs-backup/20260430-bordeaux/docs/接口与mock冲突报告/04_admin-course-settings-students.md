# 接口与 Mock 冲突：`/admin/course-settings/[courseId]/students`

## 1. 冲突等级

`中高`

这页的学生列表本身能画出来，但“今日课程名册页”的上游对象已经缺失。

## 2. mock 依赖的对象

mock 名册页依赖的是“今日生效课程名册对象”。

它不只是学生数组，还应包含：

- 当前课程的今日上下文
- 时间、地点、老师、人数
- 名单里需要高亮的临时学生或重点学生
- 与课程配置页连续的链路信息

## 3. 当前接口实际使用的对象

当前链路是：

- `fetchCourseDetail`
- `fetchAdminCourseStudents`
- `mapAdminCourseRosterData`

这本质上是：

- 课程详情对象
- 学生列表对象

它可以做“课程 roster”，但做不了“今日课程名册”。

## 4. 具体冲突点

### 4.1 对象冲突

mock 需要的是：

- 今日课程名册对象

当前拿到的是：

- 静态课程详情 + 学生列表

### 4.2 字段冲突

当前对象缺少：

- 当前课程为什么属于今天
- 当前点名老师是谁
- 当前名册是否存在临时加入学生
- 需要高亮的学生状态

### 4.3 链路冲突

mock 里这页是从“今日生效课程”进入的。

当前页从数据语义上已经与“今日课程集合”脱钩，变成通用学生列表页。

## 5. 影响面

- 页头信息变浅。
- 搜索维度收窄。
- 临时学生和重点学生状态无法表达。

## 6. 处理建议

- 这页需要补“今日课程上下文”，不能只靠 `fetchCourseDetail`。
- 即便暂时继续复用学生列表接口，也要由上游真实接口结果聚合出 `effectiveCourseContext` 一类对象。
- 高亮状态如果后端没有，需要由真实接口结果推导或新增接口字段，不能回落到 mock 状态。
