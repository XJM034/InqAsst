# 页面对比报告：`/admin/home`

## 1. 先说结论

`/admin/home` 的整体样式并没有明显跑偏，主要问题是“入口卡片仍在，但其中一部分统计语义已经变了”，最典型的就是“课程设置”入口角标。

所以这页属于：

- 视觉结构基本对
- 入口语义部分失真
- 需要跟着核心业务页一起修

## 2. mock 页面想表达什么

mock 管理员首页本质上是“今日管理总入口”，重点是让管理员快速判断今天该先点哪里：

- 今天有哪些课程要管
- 今天点名整体情况怎样
- 临时代课是否有异常
- 哪些入口优先处理

因此首页上的角标和摘要，不只是装饰，它们在表达“今天要处理什么”。

## 3. 当前接口版实际做成了什么

当前实现主要来自：

- `app/admin/home/page.tsx`
- `lib/services/mobile-app.ts` 中的 `getAdminHomeData`
- `lib/services/mobile-api.ts` 中的 `fetchAdminHomeSummary`
- `lib/services/mobile-adapters.ts` 中的 `mapAdminHomeData`

它已经变成真实总览页，摘要卡片取值来自后端汇总接口，而不是 mock 里手工控制的首页数据模型。

## 4. 页面展示差异

### 4.1 外观版式基本仍在

- 顶部欢迎信息还在
- 常用入口卡片还在
- 今日摘要卡还在

所以用户一眼看到的骨架不会觉得“完全换了一页”。

### 4.2 “课程设置”入口角标意义偏了

mock 里的这个角标，表达的是“今日生效课程数量”。

当前接口版里，这个数字是通过首页汇总数据推导出来的，表达更接近：

- 点名规则条数
- 时间窗口配置数
- 或者某种规则统计

这会导致首页虽然长得像原来，但管理员点进去前看到的预期已经错了。

### 4.3 首页与子页的语义不再严格联动

mock 的首页入口是和子页面含义严格配套的。

当前情况是：

- 首页入口名还叫“课程设置”
- 但对应页已经不是 mock 那个“今日生效课程配置页”
- 入口角标也不再表达 mock 语义

于是首页和子页一起漂移了。

## 5. mock 与实际接口数据差异

### 5.1 mock 数据来源

mock 使用：

- `adminHomeData`
- `adminHomeDataByCampus`

它可以精确控制每个入口卡片要表达什么。

### 5.2 当前接口数据来源

当前使用：

- `getAdminHomeData`
- `fetchAdminHomeSummary`
- `mapAdminHomeData`

它依赖的是后端汇总字段，而不是“为首页设计出来的专用 UI 语义字段”。

### 5.3 关键缺口

当前首页汇总数据没有天然提供 mock 所需要的这些精确信息：

- 今日最终生效课程数
- 今日需要临时代课处理的课程数
- 与 mock 子页面一一对应的入口语义字段

因此 adapter 只能“找近似字段替代”，这正是首页语义跑偏的原因。

## 6. 对齐建议

- 重新定义首页入口 badge 的来源，不要继续拿 `rollCallWindowRuleCount` 之类的近似字段硬凑。
- 让首页入口统计回到 mock 语义，也就是和对应子页的真实列表结果对齐。
- 如果后端没有“今日生效课程数”这类字段，应该从课程设置数据源统一推导，而不是在首页单独猜。
- 首页的优先级提示要和 `/admin/course-settings`、`/admin/emergency`、`/admin/control` 保持同一套日视角。

## 7. 对齐时要重点核对的点

- “课程设置”角标是不是表达“今日生效课程数”。
- “老师设置/代课处理”类入口是不是表达当天待处理项，而不是周维度统计。
- 首页摘要数字点进去后，是否能在对应页面找到一一对应的列表结果。
