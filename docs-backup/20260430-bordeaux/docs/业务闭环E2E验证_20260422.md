执行口径：本机前端 `http://127.0.0.1:3000` + shared dev `proxy` 后端

## 1. 本轮结论

- 本轮业务闭环 **未通过发布验收**。
- 标准点名主链已跑通：
  - 管理端设置实际上课时间
  - 管理端设置点名时间
  - 老师端进入可提交窗口
  - 老师端提交点名
  - 管理端总控 / 未到学生 / 复制未到学生同步
- 系统外老师补充链已跑通：
  - 录入系统外老师
  - 课程详情页切换为系统外老师
  - 更换点名老师页可复用该系统外老师
  - 恢复默认老师成功
- 系统内代课老师闭环当前被 shared dev 后端阻塞：
  - UI 选择系统内老师时返回固定冲突
  - API 直接探测多名系统内老师，均返回同一条冲突信息
  - 因此当前不能宣称“管理端设置系统内代课 -> 代课老师登录进入 -> 完成点名”可用
- 2026-04-22 晚些时候前端已继续接入“同日系统内老师批量互换”入口与 batch 契约：
  - 前端已改为继续沿用单课“更换点名老师”流程，在选择到同时间段已有课的系统内老师时提示是否互换课程
  - 前端会调用 `POST /api/admin/roll-call-teacher/batch/options` 与 `PUT /api/admin/roll-call-teacher/batch`
  - 但本文件当前还没有 shared dev live 证据，因为后端尚未提供可验收的 batch 最终态校验能力

  - 例如 `丁群 · 13990691184`
  - `campusId=147`

- 同校区且当天无课：
  - 对样本B保存 `何彤雯 · 18113308350`
  - 返回：
    - `code=1`

- 跨校区且当天无课：
  - 对样本B保存 `丁群 · 13990691184`
  - 返回同一条固定冲突
  - 对样本C保存 `丁群 · 13990691184`
  - 也返回同一条固定冲突
- 同校区且真实重叠：
  - 对样本B保存 `何伟 · 15928073530`
  - 仍返回同一条固定冲突

UI 侧补充确认：

- 在样本B“更换点名老师”页搜索并选择同校区无课老师 `何彤雯 · 18113308350`
- 确认更换后，弹窗内错误文案为：
  - `老师「魏雅婷」在 2026-04-22 16:45-17:45 已有排课，暂时不能设为当前点名老师`
- 同时出现兜底入口：
  - `改录系统外老师`
- 这说明当前 UI 层已把错误翻译成中文，但错误对象仍不是当前被选老师

新增结论：

- 当前 shared dev 至少不是“稳定可复现地支持系统内代课老师保存”
- 在已复测的样本B / 样本C，以及当天首批 10 个课节、同校区候选子集里，未找到稳定成功正例
- 当前更准确的现象是：
  - 同校区且真实重叠的老师，部分会返回自己的真实冲突
  - 同校区或跨校区且当天无课的老师，部分会被错误命中为 `魏雅婷 2026-04-22 16:45-17:45` 冲突
- 因此当前 `TC-06A`、`TC-06B`、`TC-06E`、`TC-06F`、`TC-06G`、`TC-06H` 仍未在这轮复测中进入成功闭环
- 若业务不允许跨校区代课，候选列表跨多校区暴露本身也是一条待确认风险

## 5. 当前缺陷与影响

### 5.1 阻塞缺陷：系统内代课老师无法保存

- 影响链路：
  - 管理端“更换点名老师 -> 选择系统内老师”
  - 代课老师登录进入
  - 代课老师完成点名
- 影响判断：
  - 当前系统内代课闭环不能验收通过
  - 这条问题应继续记入 `docs/backend-collab/current-issues.md`
- 2026-04-22 前端补充优化：
  - 冲突错误已改成中文可读提示
  - 搜索后默认老师信息会继续保留
  - 冲突出现时，弹窗内可直接跳“录入系统外老师”兜底链路
  - 以上优化不改变当前后端阻塞结论
