# 前端联调业务口径说明

更新时间：2026-04-10

本文档不是接口文档，而是本轮已经确认的业务规则说明。  
前端页面语义、状态展示、分组逻辑、切换逻辑，请以本文为准。

## 1. 多校区口径

### 1.1 登录阶段

多管理员手机号的标准流程：

1. 调 `/api/auth/login`
2. 若返回 `selectionRequired=true`
3. 展示 `campusOptions`
4. 调 `/api/auth/select-campus`
5. 拿正式 token 进入业务页

### 1.2 登录后切校区

当前已支持：

- 管理员登录后切换校区
- 切换时后端会重新签发一张新的正式 token

前端需要注意：

- 不能只改页面 query 参数
- 必须调用后端切校区接口获取新 token
- 切换成功后，前端要替换本地旧 token

### 1.3 query `campus` 的含义

已确认口径：

- 页面 query 参数里的 `campus` 仍然视为前端展示态
- 真实权限切换必须以后端新 token 为准

## 2. 点名状态口径

本轮统一口径：

- `0`：未到
- `1`：请假
- `2`：已到

补充说明：

- 老师端和管理端共用同一套状态枚举
- 不需要“未点名 / 待确认 / 补录”这类中间状态
- `null` 仅表示当前还没有该学生的点名结果，不属于状态枚举本身

前端展示建议：

- `0` -> 未到
- `1` -> 请假
- `2` -> 已到
- `null` -> 未点名

## 3. 学生状态与课节关系

已确认：

- 学生点名状态与课节挂钩
- 同一个学生在不同课节的状态可以不同

这意味着前端做以下页面时，应优先带 `courseSessionId`：

- 点名名单页
- 最近点名记录
- 未到学生页
- 管理端课程学生名单页

否则取到的只是“课程最近一次记录”，不一定是你当前看的那节课。

## 4. 未到学生页口径

已确认：

- 不是只看“今天所有未到”
- 是看“对应课程 / 对应课节”的未到与请假学生
- 分组口径：按课节分组
- 不按课程分组
- 需要展示行政班信息
- 需要展示老师信息
- `courseSessionId` 需要作为关键标识

前端理解建议：

- 这一页本质上是“课节维度的非已到学生列表”
- `status=0` 是未到
- `status=1` 是请假
- 本页不应把请假和未到混成同一颜色语义，除非设计明确要求

## 5. 总控页进度口径

已确认：

- 进度按学生口径算
- 不是按课程数量算
- 不是按课节数量算
- 当前后端返回的 `progressPercent` 口径是：
  - `已到人数 / 应到人数 * 100`

补充说明：

- 当前不需要返回“已提交 / 未提交 / 部分提交 / 异常”这类过程状态
- 当前更关注课节下学生的最终点名结果统计
- 老师姓名不是总控主返回字段中的必需项

前端展示建议：

- 用 `shouldAttendCount / presentCount / leaveCount / absentCount / progressPercent`
  做统计卡和进度条
- 不要自行再推导另一套百分比口径

## 6. 时间设置口径

### 6.1 默认规则

已存在的 `campus_roll_call_window` 代表：

- 校区级默认点名时间模板规则

它的作用是：

- 给课节时间设置提供默认推导值

### 6.2 课节级设置

已确认：

- 编辑是针对课节生效
- 不是按整个校区立即统一覆盖
- 改某一个课节，不影响其它课节

已支持：

- 实际上课时间
- 点名时间
- 恢复默认

前端理解建议：

- “默认规则”来自校区模板
- “最终生效值”优先来自课节级配置
- 若课节未被单独覆盖，则前端会看到后端返回的推导默认值

### 6.3 picker 页含义

已确认：

- picker 页不是纯只读
- 可以作为选择器用于编辑时间

## 7. 学生新增 / 编辑 / 导入

已确认：

- 本轮要做 live
- 新增 / 编辑需要做课程、行政班、校区联动校验

尚未最终确定：

- 导入是文件上传还是粘贴文本

因此前端当前可以先按下面理解推进：

- 新增 / 编辑：按真实接口方向设计
- 导入：交互方案还需要前端和产品最终定稿

## 8. 当前最重要的前端使用原则

### 8.1 不要把“展示态切换”当成“权限切换”

例如：

- query 参数变化
- 本地选中校区变化
- 页面 tab 切换

这些都不能替代真实后端 token 切换。

### 8.2 凡是涉及点名状态，优先使用课节维度

特别是：

- 名单页状态
- 最近点名记录
- 未到学生页
- 总控详情

尽量带 `courseSessionId`，否则会出现“看的是 A 课节，拿的是 B 课节最近状态”的错位。

### 8.3 前端展示文案和筛选值分开

建议：

- 接口值继续用 `0 / 1 / 2`
- 页面展示再转成：
  - 未到
  - 请假
  - 已到

不要把展示中文直接回写成接口值。

## 9. 前端可直接参考的固定结论

前端可以直接按下面这些结论编码，不需要再等待：

- 多管理员手机号要先选校区
- 登录后管理员可切校区，但必须换新 token
- 点名状态统一为 `0 / 1 / 2`
- 不存在中间状态
- 学生状态按课节维度处理
- 未到学生页按课节维度理解
- 总控页进度按学生口径理解
- 时间设置以课节级生效为准
- 默认规则只是模板，不是最终生效值
