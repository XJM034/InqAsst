## 五、老师端页面说明

### 5.1 老师端首页

#### 5.1.1 页面结构

- 顶部问候：`李老师，下午好`
- 周次区：`第3周`
- 周历区：`一 / 二 / 三(今天) / 四 / 五`
- 今日行程标题区：
  - 标题：`今日行程`
  - 强调标识：`优先`
- 任务卡片（今日示例课程）：
  - 校区：`嘉祥成华小学`
  - 时间：`16:30 - 17:30`
  - 课程：`无人机飞行 A班`
  - 地点：`科技楼 > 2层 > 205教室`
- 明日预览区：
  - 标题：`明日行程`
  - 内容示例：`锦江校区 · 16:30 无人机飞行 A班`
- 操作按钮：
  - `开始点名`
  - `联系管理老师`
- 底部导航：`首页 / 考勤 / 我的`

#### 5.1.2 关联弹层

- 学生名单弹窗（独立原型页）：
  - 标题示例：`无人机飞行 A班 名单`
  - 字段：应到人数、学生姓名、行政班级、班主任、标签信息

### 5.2 老师端考勤页（课程列表）

- 页面标题：`考勤`
- 日期文案：`今天 2026-02-05 (周四)`
- 课程卡片 1（示例）：
  - `乐高机器人 A班`
  - `嘉祥成华小学 | 科技楼205 | 16:30-17:30`
  - 统计：`已到20 / 请假2 / 缺勤1 / 未点2`
  - 按钮：`开始点名`
- 课程卡片 2（示例）：
  - `足球 U8班`
  - 状态文案：`课前仅可请假`

### 5.3 老师端点名页

- 顶部标题：`乐高机器人 A班 点名`
- 汇总信息：
  - `应到：25人`
  - `已到20 / 请假2 / 缺勤1 / 未点2`
- 快捷筛选：
  - `全部 / 仅看未点名`
- 学生项信息（原型示例）：
  - 姓名、行政班级
  - 标签或历史报课提示（如有）
  - 管理端修订标识（直接显示在对应学生考勤状态下方）
  - 当前状态标签（如 `未点 / 已到 / 请假 / 缺勤`）
- 编辑时段提示：`上课时段内可继续编辑，课后由管理员补录`
- 时段规则：`课前仅可请假；上课中可编辑；课后仅管理员补录至23:59:59`
- 底部按钮：`完成点名（未点2）`

### 5.4 点名提交弹窗（3态）

- 状态 A（未点名阻断态）：
  - 触发条件：`未点名人数 > 0`
  - 标题示例：`还有2名学生未点名`
  - 内容：展示未点名姓名（如 `张三、王五`）
  - 提示：`请先给所有学生选择状态后再提交`
  - 按钮：仅 `继续点名`
- 状态 B（缺勤/请假确认态）：
  - 触发条件：`未点名人数 = 0` 且 `缺勤+请假 > 0`
  - 标题示例：`请确认以下学生状态`
  - 内容：分组展示缺勤/请假人数与姓名
  - 按钮：`修改点名` + `完成点名`
- 状态 C（全勤确认态）：
  - 触发条件：`未点名=0 且 缺勤=0 且 请假=0`
  - 标题示例：`班级全勤`
  - 内容示例：`本节课应到25人，实到25人`
  - 按钮：`修改点名` + `完成点名`
- 完成判定口径：`已完成 = 未点名人数为0`（与是否点击「完成点名」无关）

### 5.5 老师端我的页

- 信息区：
  - 姓名：`李老师`
  - 手机：`138****1234`
  - 角色：`外部老师`
- 操作按钮：`退出登录`
- 交互提示：`点击退出后将二次确认`

## 六、管理端页面说明

  - `16:30 - 17:30`
  - `授课教师：王力`
  - 地点：`科技楼 > 2层 > 205教室`

#### 6.1.2 搜索预览列表（独立原型页）

- 输入能力（原型文案表达）：
  - 学生、老师、课程均支持：汉字 / 拼音全拼 / 拼音首字母
  - 用户输入字符时触发即时联想（Type-ahead）
  - 可同时匹配学生库、教师库、课程库
- 输入示例：`l`
- 结果示例：
  - 学生：李景行（5年级3班 | 班主任：王某某）
  - 学生：李京星（4年级8班 | 班主任：刘某某）

- 检索示例说明：
  - `ljx`：可命中学生/老师

#### 6.1.3 结果详情卡片展示（独立原型页）
