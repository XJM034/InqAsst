## 1. 背景

当前前端已完成以下主链 live 对接：

- 登录 / 选校区 / 切校区
- 老师点名
- 管理端点名总控
- 未到学生页
- 课节级时间设置

当前剩余的主要联调阻塞项，集中在课程学生名单的“新增 / 编辑 / 导入”。

本文件只写明确缺口，不写模糊建议，供后端直接评估接口补充范围。

## 2. 学生新增接口需求

对应页面：

- `/admin/course-settings/[courseId]/students/new`

前端当前需要支持的业务动作：

1. 创建一个学生
2. 让这个学生进入当前课程名单
3. 需要同时满足校区、课程、行政班校验

后端需要明确的契约：

### 2.1 创建入口责任边界

请明确以下两种方案中的哪一种是最终口径：

#### 方案 A：单接口完成

- 一个接口同时完成：
  - 创建学生
  - 加入课程名单

#### 方案 B：拆成两个动作

- 接口 1：创建学生
- 接口 2：把学生加入课程名单

如果采用方案 B，请明确两个动作的调用顺序与失败回滚口径。

### 2.2 新增接口所需字段

前端至少需要明确以下字段是否必填：

- `name`
- `campusId`
- `homeroomClassId`
- `courseId`
- `externalStudentId`（如果有）
- 其他业务唯一标识字段（如果有）

请逐字段给出：

- 字段名
- 类型
- 是否必填
- 中文语义
- 校验规则
- 是否允许为空字符串 / `null`

### 2.3 新增成功返回值

前端需要新增成功后立即刷新课程名单，因此请明确返回值至少包含：

- `studentId`
- `studentName`
- `homeroomClassId`
- `homeroomClassName`
- `courseId`（若接口已绑定课程）

如果后端希望前端新增后重新拉课程名单，也请明确是否只返回：

- 新建学生主键
- 或 `null`

## 3. 学生编辑接口需求

对应页面：

- `/admin/course-settings/[courseId]/students/[studentId]/edit`

前端当前需要支持的业务动作：

1. 进入页面时读取单学生详情
2. 修改学生姓名、行政班等基础信息
3. 明确课程名单关联是否允许在此页同时调整

后端需要明确的契约：

### 3.1 单学生详情读取接口

前端需要一个明确的读取接口，至少返回：

- `studentId`
- `studentName`
- `campusId`
- `homeroomClassId`
- `homeroomClassName`
- `externalStudentId`（若有）
- 当前是否已在某课程名单中（若编辑页承载此职责）

### 3.2 单学生更新接口

请明确更新接口支持哪些字段修改：

- `name`
- `homeroomClassId`
- `externalStudentId`
- 其他字段

并明确：

- 哪些字段可改
- 哪些字段不可改
- 更新时是否校验“学生校区必须与当前管理员校区一致”

### 3.3 课程名单关联的责任边界

这是当前最需要后端确认的点。

请明确以下哪种是最终责任边界：

#### 方案 A：学生编辑接口只负责学生主数据

- 课程名单关联不在本接口处理
- 若需要变更课程名单，另走课程 roster 接口

#### 方案 B：学生编辑接口允许同时处理课程名单关联

- 编辑学生时可同步把该学生加入 / 移出当前课程

请不要保持模糊状态，需要给出单一最终口径。

## 5. 可选增强项：可切换校区列表接口

当前前端默认实现：

- 登录阶段拿到 `campusOptions`
- 本地持久化后，用于“我的”页切校区入口

如果后端不希望前端依赖本地持久化，请补一个可选接口：

- 获取当前登录手机号 / 账号可切换的管理员校区列表

前端期望字段：

- `campusId`
- `campusName`
- `adminUserId`
- `adminName`

## 6. 希望后端确认的最终事项

请后端至少回复并确认以下 4 件事：

1. 学生新增是“单接口完成”还是“创建 + 加名单拆两步”
2. 学生编辑是否只改学生主数据，还是允许同时调整课程名单关联
3. 单学生详情读取接口的最终路径与返回字段
4. 是否需要新增“可切换校区列表”查询接口，替代前端本地持久化 `campusOptions`
