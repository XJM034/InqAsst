## 问题陈述

K12 课后延时教育公司面临的核心痛点：
1. **走班制下师生找不到上课地点** - 外部老师不熟悉校园,学生在多个校区间流动
2. **考勤管理分散低效** - 老师现场点名,管理员需要汇总统计,缺乏实时可见性
3. **信息碎片化导致管理混乱** - 管理员面对迷路学生/未带工牌老师,需要在碎片化线索下快速定位

本次设计基于甲方最新反馈,回到双角色架构(老师端 + 管理端),聚焦 P0 核心功能:查询和考勤。

## 核心设计理念

- **极简信息架构** - 老师端是"智能行程单",管理端是"全能搜索与管控台"
- **高容错率** - 支持拼音、首字母、同音字模糊搜索,应对信息碎片化场景
- **上帝视角** - 管理端可全局查询学生/老师/课程,快速"破案"

## 约束条件

1. **角色架构**: 本期只做双角色(老师 + 管理员),v2 的助教端、通知系统、推送系统全部延后
2. **校区权限**: 管理员单校区绑定不可切换;老师可跨校区,按天自动显示当天所在校区
3. **移动端优先**: 设计针对手机端,需考虑屏幕空间利用率
4. **技术边界**: 设计文档只描述需求和交互,不涉及具体技术实现方案

## 前提条件

1. 本期回到双角色架构,v2 的三角色架构(助教端、通知系统、推送系统)全部延后
2. 搜索功能需要支持同音字匹配(如"李嘉树"匹配"李佳澍"),后端需将姓名转换为拼音存储
3. 考勤时间权限明确:老师端只能在上课时间段内点名,管理端可在课程开始后至次日 23:59:59 修改"请假"状态
4. 学生名单布局从"一行一人"优化为"网格卡片布局(每行 2-3 人)",提高空间利用率
5. 所有"导出"功能指复制到剪贴板,而非生成文件下载

## 考虑的方案

### 方案 A: 最小化调整(基于 v1.7 微调)

**概述**: 在现有 v1.7 架构基础上进行局部优化,保持整体结构不变,只调整具体交互细节。

**核心变更**:
- 搜索功能增加同音字匹配能力
- 学生名单布局从列表改为网格(每行 2-3 人)
- 管理端考勤增加两个导出功能(复制到剪贴板)
- 删除学生个人特征标签

**优点**:
- 开发工作量最小,风险可控
- 与现有原型图差异小,易于理解和验收
- 可快速交付,满足甲方时间要求

**缺点**:
- 未从根本上优化信息架构,仍有改进空间
- 部分交互细节(如导出格式)需要在开发中细化

**工作量**: S (小型调整,预计 1-2 周开发周期)

**风险**: 低

**复用**: 完全基于现有 v1.7 设计,复用所有页面结构和组件

### 方案 B: 结构化重构(优化信息架构)

**概述**: 在满足甲方需求的同时,对信息架构进行系统性优化,提升整体用户体验。

**核心变更**:
- 管理端考勤页面重新设计:统计概览 + 班级列表 + 学生名单三层结构
- 老师端考勤增加快捷筛选(全部/已到/未到)和倒计时提醒
- 搜索结果优先级调整:学生 > 老师 > 课程
- 导出功能标准化:统一格式,支持微信群直接粘贴

**优点**:
- 信息层级更清晰,符合管理员和老师的实际工作流
- 考勤统计可见性更强,管理员可快速掌握全局状态
- 导出功能更实用,减少管理员手工整理工作

**缺点**:
- 需要重新设计部分页面结构,开发工作量增加
- 与现有原型图差异较大,需要重新评审

**工作量**: M (中型重构,预计 2-3 周开发周期)

**风险**: 中

**复用**: 复用现有组件和交互模式,但页面结构需要调整

## 推荐方案

**推荐方案 A: 最小化调整**

**理由**:
1. **时间优先** - 乙方接单项目,甲方已经过多轮需求对齐,现在需要快速交付而非大幅优化
2. **风险可控** - 基于 v1.7 微调,开发团队熟悉现有架构,不会引入新的技术风险
3. **需求已明确** - 甲方反馈非常具体,都是局部调整而非架构性变更,方案 A 完全覆盖
4. **可迭代** - 本期聚焦 P0 功能快速上线,后续可基于用户反馈迭代优化

方案 B 虽然在信息架构上更优,但会增加开发周期和沟通成本,不符合当前项目阶段的目标。

## 详细设计规范

### 一、全局规范

#### 1.1 角色与权限

**角色定义**:
- **老师端(Teacher)**: 外部上课老师,查看自己的课程行程,现场点名
- **管理端(Admin)**: 校区管理老师,全局搜索查询,考勤总控

**校区权限规则**:
- 管理员登录后只能看到自己所属校区的数据,不可切换校区
- 老师可能在不同校区上课,系统根据当天课程自动显示对应校区信息
- 所有页面默认显示当天(周一至周五)信息

#### 1.2 视觉规范

**地点格式标准**: 统一采用层级化文本 `楼栋 > 楼层 > 教室号`
- 示例: **综合楼 A 栋 > 3 层 > 3年级 2 班**
- 字号需醒目,作为视觉焦点

**五色分类系统**: 课程类别采用颜色标识
- 科创(蓝色系) / 人文(青色系) / 体育(橙色系) / 劳作(绿色系) / 艺术(紫色系)
- 用于课程卡片背景色、类别图标等视觉元素

**标签系统**: 删除学生个人特征标签(如"调皮""敏感"等),保留课程类别标签

#### 1.3 底部导航

- **老师端**: 首页 / 考勤 / 我的
- **管理端**: 首页 / 名单 / 我的

### 二、老师端设计

#### 2.1 首页(Teacher_Home) - P0 功能

**定位**: 智能行程单,系统根据账号直接告知"何时、去哪、哪个校区"

**页面结构**:

**A. 顶部导航区**
- 左侧: 用户头像 + 欢迎语(如"李老师,下午好")
- 右侧: 周历切换器(周一至周五横向排列)
  - 默认选中"今天"
  - 有排课的日期下方显示蓝色圆点标记
  - 点击不同日期,下方任务卡片内容刷新

**B. 核心任务卡片**

若当日有课,显示任务卡片(支持多张堆叠):

1. **卡片眉部**: 通栏深色标签,醒目显示 **📍 [校区名称]** (如:成华校区)
2. **卡片主体**:
   - 背景色根据课程类别显示淡淡的主题色
   - 第一行: **时间**(16:30 - 17:30,特大字号加粗) + **课程名称**(无人机飞行 A 班) + 类别图标
   - 第二行(视觉焦点): 独立色块容器,显示 **楼栋 > 楼层 > 教室**,右侧预留导航图标按钮
3. **卡片底部**:
   - ~~删除"查看班级名单"按钮~~ (本期不提供)
   - 保留"联系管理老师"按钮(一键拨号)

**C. 空状态**
- 若选中日期无课,显示插画及文案:"该日无教学安排"

#### 2.2 考勤页(Teacher_Attendance) - P0 功能

**定位**: 老师现场点名工具,快速完成学生考勤

**页面结构**:

**A. 课程列表**
- 显示当日所有课程,按时间排序
- 每个课程卡片显示:时间段 + 课程名称 + 地点 + 点名状态
- 点名状态:
  - 课前:显示"即将开始"
  - 上课时间段内:显示"开始点名"按钮
  - 课后:显示"已完成"或"查看详情"

**B. 点名页**

进入点名页后:

1. **顶部倒计时提醒条**:
   - 显示"请在 X 分钟内完成点名并提交"
   - 上课后开始倒计时,5 分钟内为正常提示(蓝色/灰色)
   - 超过 5 分钟后提示条变红色警告
   - 不阻断提交,只是视觉提醒

2. **快捷筛选按钮**:
   - 三个按钮:全部 / 已到 / 未到
   - 默认选中"全部"
   - 点击切换筛选,下方学生列表实时刷新

3. **学生名单(网格卡片布局)**:
   - 每行显示 2-3 个学生卡片
   - 每个卡片显示:姓名 + 行政班级
   - 删除头像和学生个人特征标签
   - 考勤状态:未到(默认) / 已到
   - 点击卡片切换状态(未到 ↔ 已到)

4. **底部统计与提交**:
   - 实时统计:应到 X 人 / 已到 Y 人 / 未到 Z 人
   - "提交考勤"按钮

**C. 提交弹窗(三态逻辑)**

根据点名结果显示不同弹窗:

1. **未点名阻断态**(未点名人数 > 0):
   - 标题:"还有 X 人未点名"
   - 文案:"请确认所有学生考勤状态后再提交"
   - 按钮:仅"继续点名"(关闭弹窗)

2. **全勤确认态**(未点名人数 = 0 且未到人数 = 0):
   - 标题:"全员到齐"
   - 文案:"本节课应到 X 人,已全部到齐"
   - 按钮:"确认提交" + "返回修改"

3. **缺勤确认态**(未点名人数 = 0 且未到人数 > 0):
   - 标题:"确认提交考勤"
   - 文案:"本节课应到 X 人,已到 Y 人,未到 Z 人"
   - 按钮:"确认提交" + "返回修改"

**时间权限规则**:
- 只能在上课时间段内进行点名
- 课前和课后均不可修改考勤状态

#### 2.3 我的页(Teacher_Profile)

- 用户信息:头像、姓名、所属机构
- 设置入口:修改密码、退出登录等
- 删除标签管理相关功能

### 三、管理端设计

#### 3.1 首页(Admin_Home) - P0 功能

**定位**: 全能搜索与管控台,基于"上帝视角"快速定位学生/老师/课程

**页面结构**:

**A. 超级搜索框**

1. **搜索输入框**:
   - Placeholder:`搜学生、老师姓名、课程...`
   - 支持输入:汉字、拼音全拼、拼音首字母
   - **同音字匹配**:输入"李嘉树"能匹配到"李佳澍"(拼音相同)

2. **即时联想预览列表**:
   - 输入过程中触发即时联想
   - 键盘上方弹出半屏列表
   - 同时匹配学生库、教师库、课程库
   - **默认优先显示学生结果**(大多数场景是搜索学生)
   - 列表项带身份标签:
     - `[学生]` **李景行** —— 5年级 3 班
     - `[老师]` **王力** —— 无人机飞行课程
     - `[课程]` **乐高机器人 A 班** —— 16:30 / 科技楼 205

**B. 级联筛选器**(辅助缩小"找学生"的范围)

1. **第一排:课程类别**
   - `全部` `体育` `科创` `艺术` `劳作` `人文`

2. **第二排:年级**
   - `一年级` `二年级` `三年级` ... `六年级`

3. **第三排:行政班级**(动态显示)
   - 默认状态:隐藏或置灰
   - 激活逻辑:当用户在第二排选中某个年级后,第三排点亮并显示该年级下的班级

**C. 结果详情卡片**

根据选中对象类型(学生/老师/课程),动态渲染三种卡片:

**布局一:学生卡片**
- 左侧:学生照片(大图,点击可放大),若无照片显示性别图标
- 右侧上部(身份确权):
  - 姓名 + 行政班级
  - 校验信息:班主任姓名(用于口头核对:"你班主任姓什么?")
  - ~~删除生日等其他校验信息~~
- 右侧下部(核心指引):
  - 课程名称(带颜色标签)
  - 地点:**楼栋 > 楼层 > 教室**(醒目显示,红色加粗大字号)

**布局二:老师卡片**
- 左侧:教师照片或通用图标
- 右侧上部:
  - 姓名 + 身份(某某机构签约教师)
  - 电话图标(点击直接拨号)
- 右侧下部:
  - 当前状态:`正在上课` 或 `即将开始`
  - 课程名称
  - 地点:**楼栋 > 楼层 > 教室**

**布局三:课程卡片**
- 左侧:类别图标(齿轮/画板/足球等),背景为该类别标准色
- 右侧上部:
  - 课程名称 + 时间段
  - 班级人数:应到 X 人(点击弹出学生名单,用于点名/核对)
- 右侧下部:
  - 授课教师姓名(带电话图标)
  - 地点:**楼栋 > 楼层 > 教室**

#### 3.2 名单页(Admin_Roster)

**说明**: 本期保留名单页入口,但具体功能待后续迭代明确。当前主要通过首页搜索完成学生/课程查询。

#### 3.3 考勤页(Admin_Attendance) - P0 功能

**定位**: 考勤总控台,管理员掌握全局考勤状态并进行补录

**页面结构**:

**A. 顶部统计概览**
- 显示:已完成点名 X 班 / 未完成点名 Y 班
- 醒目展示,一目了然

**B. 导出功能区**

两个导出按钮(复制到剪贴板):

1. **导出未完成点名班级信息**:
   - 格式示例:
     ```
     【未完成点名班级】2026-03-23
     1. 乐高机器人 A 班 - 王力老师
     2. 足球 U8 班 - 张教练
     ```
   - 用途:复制后粘贴到微信群,催促老师完成点名

2. **导出未到学生信息**:
   - 格式示例(按行政班级分组):
     ```
     【未到学生名单】2026-03-23

     【5年级3班】
     - 李景行 | 乐高机器人 A 班 | 科技楼 A 栋 > 2 层 > 205

     【4年级8班】
     - 王小明 | 足球 U8 班 | 北操场 3 号区
     ```
   - 用途:复制后粘贴到班主任群,让班主任回复未到情况

**C. 班级列表**

显示当日所有课程班级:
- 每个班级卡片显示:班级名称 + 老师姓名 + 点名状态
- 点名状态:
  - 未完成(红色/橙色提示)
  - 已完成(绿色/灰色)
- 点击班级卡片 → 进入该班学生名单

**D. 学生名单(网格卡片布局)**

点击某个班级后进入学生名单:

1. **学生卡片布局**:
   - 每行显示 2-3 个学生卡片
   - 每个卡片显示:姓名 + 行政班级
   - 删除头像和学生个人特征标签

2. **考勤状态**:
   - 老师端提交的状态:已到 / 未到
   - 管理端可将"未到"修改为"请假"
   - 状态切换:点击卡片切换状态

3. **时间权限规则**:
   - 管理端可在课程开始时间之后至次日 23:59:59 修改"请假"状态
   - 次日 00:00 之后,考勤记录变为只读

#### 3.4 我的页(Admin_Profile)

- 用户信息:头像、姓名、所属校区
- 设置入口:修改密码、退出登录等

### 四、典型交互场景

#### 场景一:管理员帮助迷路学生(学生模糊查找)

1. **线索**:学生口齿不清说叫"li jing xing",看起来个子挺高
2. **操作**:管理员输入 `ljx`
3. **初筛**:预览列表优先显示学生结果,出现 `李景行(5年级)` 和 `李京星(1年级)`
4. **判断**:管理员根据身高,点击 `李景行(5年级)`
5. **校验**:卡片弹出,管理员看照片确认长相,并问:"你班主任姓什么?" 学生答:"姓王" —— **身份确认**
6. **结果**:管理员指引:"去科技楼 205 教室"

#### 场景二:管理员帮助未带工牌的老师(老师查找)

1. **线索**:遇到一位陌生老师,自称姓王,来上"飞机课"
2. **操作**:管理员输入 `王`
3. **初筛**:预览列表出现多位姓王的学生和老师,管理员点击 `[老师] 王力 - 无人机课程`
4. **结果**:卡片显示该老师需前往 **科技楼 205**
5. **扩展**:如果老师迟迟未到,管理员可直接点击卡片上的电话图标进行催促

#### 场景三:管理员巡课或家长询问(课程查找)

1. **线索**:家长问"今天的足球课在哪上?"或管理员想去足球课巡查
2. **操作**:管理员输入 `足球`
3. **初筛**:预览列表显示 `[课程] 足球 U8 班` 和 `[课程] 足球 U10 班`
4. **选择**:点击 `U8 班`
5. **结果**:卡片显示 **北操场 3 号区**,以及 **张教练** 的电话。管理员可直接点击人数链接查看学生名单是否到齐

#### 场景四:老师自查行程(行程规划)

1. **操作**:老师周一晚上在家打开 App
2. **查看**:点击顶部周历的"周二"
3. **结果**:卡片显示 **明天(周二) - 锦江校区 - 16:30**
4. **决策**:老师明确了明天要去的校区和时间,提前安排出行
