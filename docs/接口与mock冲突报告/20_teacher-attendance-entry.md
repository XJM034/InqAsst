# 接口与 Mock 冲突：`/teacher/attendance`

## 1. 冲突等级

`中`

这页的冲突不在视觉，而在入口链路对象。mock 没有这层中转页，当前接口版新增了一个“根据真实 session 决定去哪里”的分发页。

## 2. mock 依赖的对象

mock 教师端底部点名入口更接近：

- 一个稳定的点名落点
- 或一个明确的 demo 入口

它不依赖额外分发对象。

## 3. 当前接口实际使用的对象

当前这页实际依赖的是：

- `getTeacherHomeData`
- 教师首页产出的当日课程结果

它的职责是：

- 找出今天可点名的课程
- 跳转到真实 `courseId/courseSessionId`
- 或进入无课页

## 4. 具体冲突点

### 4.1 链路冲突

mock 的点名入口是直接入口。

当前的点名入口变成了“首页结果分发器”。

### 4.2 上游依赖冲突

只要教师首页里的任务对象判断错了，这里就会把老师带去错误的点名页。

### 4.3 路由冲突

mock 用户无需知道真实 session 参数。

当前入口完全依赖真实实体参数。

## 5. 处理建议

- 这页可以继续存在，但必须严格建立在修正后的教师首页任务对象之上。
- 中转页只能消费真实接口聚合结果，不能自己再做第二次猜测。
- 无课判断规则也要统一来自真实接口，不允许写死或使用 mock 规则。
