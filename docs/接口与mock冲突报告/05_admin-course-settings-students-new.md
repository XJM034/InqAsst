# 接口与 Mock 冲突：`/admin/course-settings/[courseId]/students/new`

## 1. 冲突等级

`中`

这页不是数据不真实，而是数据过于真实，导致 mock 的“课程子任务表单”语义被通用实体表单替代。

## 2. mock 依赖的对象

mock 新增页依赖的是一个“课程内新增学生表单对象”。

它强调：

- 当前课程上下文
- 轻量字段
- 快速完成新增

## 3. 当前接口实际使用的对象

当前链路是：

- `fetchCourseDetail`
- `fetchAdminCampusClasses`

得到的是一个更接近真实提交表单的对象：

- 课程详情
- 校区班级列表
- 行政班 ID

## 4. 具体冲突点

### 4.1 对象冲突

mock 要的是：

- 页面上下文驱动的新增表单对象

当前拿到的是：

- 实体化录入表单对象

### 4.2 字段冲突

mock 更偏：

- 文本型、页面友好字段

当前更偏：

- `homeroomClassId`
- `homeroomClasses[]`
- 实体关系字段

### 4.3 链路冲突

mock 里用户感知是：

- 给当前课程加学生

当前更像：

- 编辑一个真实学生录入表单

## 5. 处理建议

- 保留真实字段没问题。
- 但需要在视图模型里恢复“当前课程新增学生”的语义层。
- 表单页上方必须始终带着课程上下文，而不是只剩实体字段。
