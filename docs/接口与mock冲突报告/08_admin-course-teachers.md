# 接口与 Mock 冲突：`/admin/course-teachers`

## 1. 冲突等级

`极高`

这页的问题和课程设置类似，也是上游对象拿错了。mock 需要“课程-老师关系对象”，当前接口只给了“老师实体列表”。

## 2. mock 依赖的对象

mock 课程老师页要的是“课程老师关系对象”，至少包含：

- 星期
- 课程名
- 时间
- 地点
- 当前老师
- 默认老师 / 临时代课状态

换句话说，页面核心不是老师是谁，而是：

- 老师对应哪门课

## 3. 当前接口实际使用的对象

当前链路是：

- `fetchAdminTeachers`
- `mapAdminCourseTeachersData`

接口给的主要是：

- 老师基础实体
- 姓名
- 手机号

## 4. 具体冲突点

### 4.1 对象冲突

mock 要的是：

- 课程-老师关系表

当前给的是：

- 老师通讯录

### 4.2 字段冲突

当前接口缺少页面最关键的字段：

- 课程名
- 周几
- 上课时间
- 地点
- 默认 / 临时状态

### 4.3 粒度冲突

mock 的最小单元是：

- 一位老师在某门课上的角色

当前接口最小单元却是：

- 一位老师这个实体

### 4.4 页面名与数据对象冲突

页面还叫“课程老师”，但对象却是“老师列表”，这会让页面天然失真。

## 5. 影响面

- 页面失去课程维度。
- 卡片无法承载时间地点信息。
- 代课状态无法准确表达。

## 6. 处理建议

- 这页不能继续直接消费 `fetchAdminTeachers`。
- 必须改成课程-老师关系聚合对象。
- 如果后端暂时没有对应接口，前端也要把课程、老师设置、时间地点三类数据先聚合好再渲染。
