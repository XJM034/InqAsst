# 接口与 Mock 冲突：`/admin/emergency/course/[courseId]/select-teacher`

## 1. 冲突等级

`中`

这页不是严重错页，主要冲突出在“候选老师对象是否完整”和“课节级真实提交是否压过了 mock 的课程级语义”。

## 2. mock 依赖的对象

mock 需要的是“课程老师候选对象”，至少包含：

- 当前课程摘要
- 默认老师
- 候选老师列表
- 当前已选老师
- 系统外老师入口

## 3. 当前接口实际使用的对象

当前链路是：

- `getAdminTeacherSelection(courseId, courseSessionId)`
- `fetchCourseDetail`
- `fetchRollCallTeacherOptions`

它已经是一个真实候选老师对象，但锚点是课节而不是课程。

## 4. 具体冲突点

### 4.1 粒度冲突

mock 更偏课程级老师替换。

当前是课节级老师选择。

### 4.2 字段完整性冲突

页面能否完整恢复 mock，取决于真实候选对象是否包含：

- 当前默认老师
- 当前已选老师
- 候选老师完整列表
- 备注或说明

一旦缺字段，页面虽能渲染，但细节就会塌。

### 4.3 动作冲突

mock 更像“选定后继续流程”。

当前更接近“即时提交选中结果”。

## 5. 处理建议

- 保留真实接口。
- 页面视图模型上补足 mock 需要的课程级摘要和选中态字段。
- 即时提交可留，但 UI 不要做得像运维页。
