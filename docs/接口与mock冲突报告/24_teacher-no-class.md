# 接口与 Mock 冲突：`/teacher/no-class`

## 1. 冲突等级

`中低`

这页本身没有严重字段冲突，问题在于教师端无课状态被拆成多个路由后，真实判定来源可能不一致。

## 2. mock 依赖的对象

mock 无课页依赖的其实只是一个简单空态对象：

- 今天没课
- 引导老师返回或等待

## 3. 当前接口实际使用的对象

当前页如果继续存在，就必须和教师端真实课程判断共享同一来源：

- 教师今日课程 / session 查询结果

## 4. 具体冲突点

### 4.1 链路冲突

当前教师端已有专门的点名无课页。

如果本页仍按另一套规则展示，会出现：

- 入口不同
- 无课判断不同
- 文案时机不同

### 4.2 对象统一性冲突

无课状态应是一个统一的真实判断结果，而不是两个页面各自维护的空态条件。

## 5. 处理建议

- 统一由真实教师课程查询结果驱动无课状态。
- 两个无课页的文案和落点可以不同，但判定来源必须一致。
