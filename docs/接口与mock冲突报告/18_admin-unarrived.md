# 接口与 Mock 冲突：`/admin/unarrived`

## 1. 冲突等级

`极高`

这页的分组对象完全变了。mock 是按行政班聚合，当前接口适配后按课程课节聚合。

## 2. mock 依赖的对象

mock 未到名单页需要的是“按班级聚合的未到对象”。

其最自然结构应是：

- 班级
- 该班未到学生列表
- 每个学生缺的是哪门课

这是一种“学生/班级优先”的组织方式。

## 3. 当前接口实际使用的对象

当前链路是：

- `fetchAdminAbsentStudents`
- `mapAdminUnarrivedData`

当前 adapter 的主分组键是：

- `courseId:courseSessionId`

也就是“课程课节优先”的组织方式。

## 4. 具体冲突点

### 4.1 对象冲突

mock 要的是：

- 班级聚合对象

当前用的是：

- 课节聚合对象

### 4.2 粒度冲突

mock 页面第一层最小单元是：

- 行政班

当前页面第一层最小单元是：

- 某节课

### 4.3 字段位置冲突

mock 中“缺的是哪门课”属于学生条目的说明字段。

当前因为已按课节分组，这个信息被提前挪到组头了。

### 4.4 页面目标冲突

mock 想回答的是：

- 哪个班里谁没到

当前页面回答的是：

- 某节课哪些学生异常

## 5. 处理建议

- 前端不能直接使用接口原始分组。
- 即便接口按课节返回，也应重新聚合成班级优先对象。
- 这是前端聚合层必须显式做的转换，否则页面永远回不到 mock 的浏览方式。
