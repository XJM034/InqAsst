# 接口与 Mock 冲突：`/teacher/home/roster`

## 1. 冲突等级

`中`

这页的冲突主要是链路对象变化。mock 的名单页依赖稳定演示上下文，当前页完全依赖真实 `courseId/courseSessionId`。

## 2. mock 依赖的对象

mock 名册页需要的是“当前课程名单查看对象”。

它强调：

- 当前看的是哪门课
- 现在只是看名单，不一定立刻点名

## 3. 当前接口实际使用的对象

当前页依赖：

- `getAttendanceSession(courseId, courseSessionId)`

也就是直接读取真实点名 session 对象。

## 4. 具体冲突点

### 4.1 链路冲突

mock 名册页是教师首页的自然延展页。

当前名册页必须绑定真实课节参数，已经不再是轻量延展对象。

### 4.2 上游依赖冲突

如果教师首页主课/代课判断错，这里也会进入错误课程。

### 4.3 对象边界冲突

当前名册页消费的是完整点名 session 对象，而 mock 需要的只是“名单查看对象”。

## 5. 处理建议

- 可以继续复用真实点名 session 数据。
- 但应在前端视图层显式裁剪出“名单查看对象”，不要让完整点名对象直接决定页面结构。
- 课程上下文必须来自真实 session，不允许再引入 demo/mock 兜底。
