# 接口与 Mock 冲突：`/admin/emergency`

## 1. 冲突等级

`高`

这页目前的主要冲突是：mock 要的是“今日代课任务对象”，当前使用的却是“本周老师设置总览对象”。

## 2. mock 依赖的对象

mock 需要的页面对象应聚焦：

- 今日有哪些代课课程
- 哪些课程待处理
- 哪些课程已经配置好
- featured 课程应该是哪一门

它本质上是“今日代课任务集合”。

## 3. 当前接口实际使用的对象

当前链路是：

- `getAdminEmergencyWeeklyData`
- `fetchAdminTeacherSettingsOverview(getWeekStart())`

拿到的是：

- 本周老师设置总览

## 4. 具体冲突点

### 4.1 对象冲突

mock 是：

- 今日异常 / 代课任务对象

当前是：

- 周维度老师配置对象

### 4.2 粒度冲突

mock 关注：

- 今天

当前对象关注：

- 一周

### 4.3 字段冲突

当前对象能给出的多是：

- 课程
- 老师
- 配置情况

但 mock 更关心：

- 今天是否需要代课
- 是否待处理
- featured 哪一门最优先

### 4.4 列表污染风险

只要直接消费周总览，就很容易把普通课程混进代课页。

## 5. 处理建议

- 前端不能把周总览直接渲染成“今日代课页”。
- 要么后端直接补“今日代课任务对象”。
- 要么前端先过滤出“今日且需处理”的课程，再组装页面视图模型。
