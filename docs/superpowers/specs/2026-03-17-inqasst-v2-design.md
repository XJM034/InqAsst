## 一、变更概述

### 1.1 核心变更

本次改版将系统从"考勤管理"转型为"信息查询与确认"，核心变化：

1. **考勤功能回归管理端与助教端** — 管理端新增考勤功能（与推送合并为"工作台" tab），新增助教端承担现场点名职责，老师端考勤相关页面仍移除
2. **新增助教角色** — 助教端为平行于老师端和管理端的独立角色端，聚焦课堂点名与考勤
3. **教师端聚焦信息确认** — 教师查看次日课程信息并逐课回复"已收到"
3. **管理端以学生为查询重心** — 强化学生信息查询，支持学生 → 课程、课程 → 学生双向查询
4. **标签体系改版** — 从文字标签改为颜色标记 + 内部映射规则
5. **推送系统增强** — 支持多条定时推送规则、推送模板编辑、推送对象精细选择、快速推送

### 1.2 推送机制

- 支持多条定时推送规则，每条规则可独立配置：推送模板、推送对象、推送时间、推送日期范围、启用/关闭
- 预设规则示例：`次日课程提醒`（每天 20:00，推送给次日有课的授课教师）
- 管理员可添加新的定时推送规则
- 快速推送：管理员可随时编写自定义通知内容并即时发送，支持选择推送对象
- 每条定时规则均支持"立即触发"，无需等待定时时间
- 课程信息变更处理：若推送后课程信息发生变更，管理员可手动重新推送，教师需重新确认

### 1.3 数据基础

现有数据：学生姓名、行政班、班主任姓名
验证前提：学生能写出自己姓名（无需额外身份验证机制）

## 三点五、助教端页面设计（新增角色）

助教（Assistant）为新增角色，平行于老师端和管理端。助教的核心职责是**现场课堂点名与考勤管理**，弥补老师端移除考勤功能后的空缺。

### 角色定位

| 维度 | 老师（Teacher） | 助教（Assistant） | 管理员（Admin） |
|------|----------------|-------------------|----------------|
| 核心职责 | 查看课程信息、确认通知 | 现场点名、考勤管理 | 全局信息查询、推送管理、考勤总控 |
| 考勤权限 | 无 | 上课时间段内可点名 | 课前请假 + 上课中 + 当天补录 |
| 数据范围 | 自己的课程 | 自己负责的课程 | 全校区课程 |
| 底部导航 | 首页 / 通知 / 我的 | 首页 / 考勤 / 我的 | 首页 / 名单 / 工作台 / 我的 |

### 3.5.1 助教端首页

暂参考老师端首页结构，后续可根据助教业务特点调整。

#### 页面结构

- 顶部问候：`{姓名}，{时段}好`（如 `张助教，下午好`）
- 周次区：`第3周`
- 周历区：`一 / 二 / 三(今天) / 四 / 五`
- 今日行程区：
  - 标题：`今日行程`
  - 任务卡片：校区、时间、课程名、地点
  - 操作按钮：`查看名单`（弹出学生名单弹窗）、`联系管理老师`（打开手机拨号界面）
- 明日预览区：
  - 标题：`明日行程`
  - 简要信息条：`校区 · 时间 课程名`（仅预览，不含操作按钮）
- 底部导航：`首页 / 考勤 / 我的`

### 3.5.2 助教端考勤页（课程列表）

#### 页面结构

- 页面标题：`考勤`
- 日期显示：`今天 2026-03-22 (周日)`（自动显示当天日期）

  - `开始点名` 按钮（仅上课时间段内显示）
- 底部导航：`首页 / 考勤 / 我的`

#### 课程列表项状态

| 状态 | 显示内容 | 操作 |
|------|----------|------|
| 未开始 | 显示「未开始上课」灰色文字 | 不可点名 |
| 进行中 | 显示考勤统计 + 「开始点名」按钮 | 可点名 |
| 已结束 | 显示考勤统计（只读） | 不可修改 |

### 3.5.3 助教端点名页

从考勤页课程列表的「开始点名」按钮进入。

#### 页面结构

- 顶部标题：`{课程名} 点名`（如 `乐高竞赛班5-6 点名`）
- 统计概览区：
  - `应到：25人`
  - `⚠️未点 2 / 📝请假 2 / ❌缺勤 1 / ✅已到 20`（实时统计）
  - 存在未点名学生时显示红色提示：`存在未点名学生，请尽快完成点名`
- 快捷筛选：`全部 / 仅看未到`
- 学生列表，每个学生项包含：
  - 学生姓名、行政班级
  - 颜色标签（如有）
  - 历史报课提示（如有，如 `第2次报此课程`）
  - 管理端修订标识（如有，显示在对应学生考勤状态下方：`管理员已修订`）
  - 考勤状态标签（未点 / 已到 / 请假 / 缺勤），点击切换状态
- 编辑时段提���：`上课时段内可继续编辑，课后由管理员补录`
- 底部按钮：`完成点名（未点 X）`（显示未点名人数）

#### 考勤状态说明

| 状态 | 图标 | 说明 |
|------|------|------|
| 已到 | ✅ | 学生已到场 |
| 请假 | 🔴 | 学生请假（管理端提前标记或助教标记） |
| 缺勤 | ❌ | 学生缺勤（未到且未请假） |
| 未点名 | ⚪ | 默认状态，尚未进行标记 |

#### 操作逻辑

- 所有学生默认状态为「未点名」
- 管理端提前标记的「请假」学生会显示请假状态
- 助教可以修改任何学生的状态（包括管理端标记的请假）
- 每次状态变更实时保存
- 管理端修订后，助教端在对应学生的考勤状态下方展示「管理员已修订」标识

#### 操作时机限制

- 课前：不可点名
- 上课时间段内：助教可编辑所有状态
- 课后：不可修改（由管理员补录）

### 3.5.4 点名提交弹窗（3 态）

点击「完成点名」按钮后，根据当前统计数据进入不同弹窗：

**状态 A — 未点名阻断态：**
- 触发条件：`未点名人数 > 0`
- 标题：`还有 X 名学生未点名`
- 内容：展示未点名学生姓名（如 `张三、王五`）
- 提示：`请先给所有学生选择状态后再提交`
- 按钮：仅 `继续点名`

**状态 B — 缺勤/请假确认态：**
- 触发条件：`未点名 = 0` 且 `缺勤 + 请假 > 0`
- 标题：`请确认以下学生状态`
- 内容：分组展示缺勤人数与姓名、请假人数与姓名
- 按钮：`修改点名` + `完成点名`

**状态 C — 全勤确认态：**
- 触发条件：`未点名 = 0 且 缺勤 = 0 且 请假 = 0`
- 标题：`班级全勤`
- 内容：`本节课应到 X 人，实到 X 人`
- 按钮：`修改点名` + `完成点名`

**完成判定口径：** `已完成 = 未点名人数为 0`（与是否点击「完成点名」按钮无关）

### 3.5.5 助教端我的页

- 信息区：姓名、手机（脱敏显示）、角色（`助教`）
- 操作按钮：`退出登录`（二次确认）

## 四、管理端页面设计

### 4.1 管理端首页

#### 4.1.1 页面结构

- 顶部信息：校区名、用户名
- 搜索框占位文案：`搜学生/老师/课程（汉字/全拼/首字母）`
- 搜索结果排序：学生优先 > 老师 > 课程
- 类别筛选：`全部 / 科创 / 人文 / 体育 / 劳作 / 艺术`
- 年级筛选：`全部 / 1年级 / 2年级 / 3年级 / 4年级 / 5年级 / 6年级 / 7-9年级`
- 默认结果区：课程卡片列表（课程名、时间、授课教师、地点）

#### 4.1.2 搜索预览列表

- 输入能力：学生、老师、课程均支持汉字 / 拼音全拼 / 拼音首字母
- 即时联想（Type-ahead），可同时匹配学生库、教师库、课程库
- 结果分组排序：学生 > 老师 > 课程

#### 4.1.3 结果详情卡片

- 学生卡片（v2.0 强化点：新增该学生的完整课程列表，v1.6 仅有基本信息）：
  - 姓名、行政班级、班主任
  - 颜色标签
  - 该学生报名的所有课程列表：课程名、时间、地点、授课教师
- 老师卡片：姓名、机构身份、课程与地点
- 课程卡片：课程名、时段、应到人数、授课教师、地点、学生名单入口

### 4.2 管理端名单页（改版）

#### 4.2.2 学生 tab

#### 4.2.3 课程 tab

- 课程列表项：课程名、时间、地点、授课教师、学生人数

### 4.3 管理端我的页（调整）

### 4.4 工作台页（新增）

工作台为管理端底部导航的第三个 tab，进入后顶部有 `推送 | 考勤` 两个子 tab 切换。

#### 4.4.1 推送 tab — 页面结构

- 页面标题：`推送`
- **定时推送区：**
  - 标题：`定时推送` + `添加` 按钮（右上角）
  - 推送规则列表，每条规则卡片包含：
    - 规则名称 + 启用/关闭开关
    - 推送描述：推送类型 + 推送对象范围（如 `次日课程提醒 · 推送给次日有课的授课教师`）

    - 操作按钮行：`编辑模板` / `编辑对象` / `立即触发`
  - 支持多条规则，各自独立配置和启用
- **快速推送区：**
  - 标题：`快速推送`
  - 文本输入框：`输入通知内容...`（多行）

  - 按课程类别分组（科创/人文/体育/劳作/艺术），展开显示该类别下的教师
- 按教师选择区：
  - 教师列表，支持逐个勾选
- 已选教师汇总区：
  - 展示当前已选教师列表及人数

#### 4.4.5 导出未回复名单

- 入口：管理端我的页 `导出未回复名单` 按钮，以及推送记录详情页 `复制未回复名单` 按钮
- 导出范围：当天最近一次推送中未确认的教师
- 导出格式：生成文本内容复制到剪贴板，方便直接粘贴到微信群
- 导出内容示例：
  ```
  【未回复教师名单】2026-03-17
  1. 王力 — 乐高机器人A班（16:30）
  2. 张伟 — 足球U8班（15:00）、篮球基础班（16:30）
  共2位教师未确认，请及时提醒。
  ```
- 导出后提示：`已复制到剪贴板`

#### 4.4.6 考勤 tab

考勤功能回归管理端（老师端考勤页面仍移除），管理员通过工作台的"考勤"子 tab 管理当日各课程的学生出勤情况。

**页面结构 — 考勤课程列表：**

- 页面标题：`考勤`
- 日期显示：`今天 2026-03-22 (周日)`（自动显示当天日期）
- 课程卡片列表，每张卡片包含：
  - 课程名称 + 类别标签（如 `[科创] 乐高机器人 A班`）
  - 校区 + 地点 + 时间段
  - 考勤统计行：`已到 X / 请假 X / 缺勤 X / 未点 X`
  - `班级名单` 按钮 — 进入管理端学生名单页

**考勤课程列表项状态：**

| 状态 | 显示内容 | 操作 |
|------|----------|------|
| 未开始 | 显示「课前仅可请假」灰色文字 | 不可点名，仅允许请假 |
| 进行中 | 显示考勤统计 + 「班级名单」按钮 | 可编辑考勤状态 |
| 已结束（当天） | 显示考勤统计 + 「班级名单」按钮 | 管理员可补录至 23:59:59 |
| 已结束（次日及以后） | 显示考勤统计（只读） | 不可修改 |

#### 4.4.7 管理端学生名单页（考勤编辑）

从考勤课程列表或首页课程卡片的「班级名单」按钮进入。

**页面结构：**

- 顶部标题：`{课程名} 班级名单`（如 `乐高竞赛班5-6 班级名单`）
- 统计概览区：
  - `应到：25人`
  - `已到 20 / 请假 2 / 缺勤 1 / 未点 2`
- 学生筛选器：`全部 / 未点名 / 请假 / 缺勤 / 特殊标记`
- 规则提示：
  - `课前仅可请假｜课后补录至23:59:59`
  - `状态可改：已到/请假/缺勤（不可回未点）`
  - `支持编辑：考勤状态 + 学生标签（同步给授课老师）`
- 学生列表，每个学生项包含：
  - 学生姓名、行政班级
  - 颜色标签（如有）
  - 考勤状态标签（已到 ✅ / 请假 🔴 / 缺勤 ❌ / 未点 ⚪），可点击切换
- 底部操作栏：
  - 已选人数提示：`已选 0 人`
  - `标记请假` 按钮 — 先选择学生再批量标记请假
  - `学生标签管理` 按钮 — 进入标签管理页

**考勤状态说明：**

| 状态 | 图标 | 说明 |
|------|------|------|
| 已到 | ✅ | 学生已到场 |
| 请假 | 🔴 | 学生请假（管理端提前标记或老师标记） |
| 缺勤 | ❌ | 学生缺勤（未到且未请假） |
| 未点名 | ⚪ | 默认状态，尚未进行标记 |

**管理端考勤操作规则：**

- 管理端权限高于老师端，可修改老师已提交的考勤状态
- 课前：仅允许标记请假
- 上课时间段内：管理员可编辑所有状态
- 课后当天（截止 `23:59:59`）：管理员可补录/修改
- 次日及以后：考勤记录只读
- 不允许把已标记状态改回「未点名」
- 标签编辑结果同步展示给授课老师

**完成判定口径：**

- `已完成 = 未点名人数为 0`（与是否点击某按钮无关）
- 考勤状态筛选中的「已完成」筛选项使用此口径

### 4.5 标签管理页（改版）

- 标题：`标签管理`
- 颜色标记列表：每个颜色一行，显示颜色色块 + 内部备注说明（仅管理员可见）
- 操作：编辑备注、删除颜色标记
- 底部按钮：`添加新标记`（选择颜色 + 填写内部备注）
- 映射规则说明：页面顶部提示 `颜色标记仅校区内部可见，不对外展示`

### 4.6 学生颜色标签分配

- 入口：管理端学生详情卡片（从首页搜索结果或名单页进入）
- 学生详情卡片底部显示：`标签` 区域，展示当前已分配的颜色色块
- 操作：
  - 点击 `+` 按钮弹出颜色选择器，从已配置的颜色标记中选择，点击即分配
  - 点击已有颜色色块可移除该标签
- 支持为一个学生分配多个颜色标签
- 不支持批量分配（逐个学生操作）

### 4.7 联系方式配置

- 入口：管理端我的页，新增 `联系方式设置` 功能入口
- 配置内容：校区管理老师联系电话（一个校区一个号码）
- 用途：教师端首页 `联系管理老师` 按钮调用此号码打开手机拨号界面
- 未配置时：`联系管理老师` 按钮置灰，提示 `管理老师暂未设置联系方式`

## 五、全局视觉与信息约定

### 5.1 地点展示格式

保持不变：`楼栋 > 楼层 > 教室/区域`

### 5.2 课程类别标签

保持不变：科创、体育、人文、劳作、艺术

### 5.3 颜色标记体系（替代原文字标签）

- 使用颜色色块标记学生特点，不使用文字描述
- 颜色与含义的对应关系为内部映射规则，仅校区管理员可见
- 教师端展示颜色标签但不展示含义说明
- 避免使用"调皮""敏感"等明显特殊化学生的字眼
