## 本轮结论

- 当前前端在“登录 -> 落角色首页 -> 个人页展示 -> cookie 持久化”这条链路上，和后端老师给出的这组 dev RDS 账号口径是一致的
- 现有 smoke 原本只覆盖了“登录与读链路可打开”，没有把“账号 / 角色 / 校区一致性矩阵”明确写成测试项
- 本轮已补充对应文档，并已把这类断言以可选环境变量的方式并入 live smoke，避免以后换账号时又要临时手工比对
- 当前可复用的环境变量见：
  - `.env.e2e.example`
  - `E2E_*_EXPECTED_NAME`
  - `E2E_*_EXPECTED_CAMPUS`
- 注意：
  - 这份文档证明的是“身份一致性”
  - 不等同于“当天课程 / summary / overview 等业务数据一致性”
