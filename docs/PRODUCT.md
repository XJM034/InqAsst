# 到了么产品说明

更新日期：2026-04-28

## 产品定位

到了么是面向课后服务场景的移动优先 H5/PWA，用于支持老师端点名和管理端现场总控。项目已上线并进入内测期，当前文档用于说明稳定产品边界、内测范围和后续迭代判断口径。

## 用户与角色

- 老师：查看今日课程、进入点名、提交学生到课状态；在需要时处理代课、合班、补录临时学生等现场情况。
- 管理员：查看现场总控、未到学生、点名时间、课程设置、老师设置和应急调整；处理系统内 / 系统外老师、课程学生、临时课程等管理操作。
- 多校区管理员：登录后先选择校区，再进入管理端流程。

## 当前核心范围

- 登录与身份保持：手机号登录，`inqasst_*` cookies 持久化；单角色直达，多校区管理员先选校区。
- 老师端：教师首页、点名入口、单课点名、合班点名、无课状态、个人页。
- 管理端：管理首页、总控、未到学生、点名时间、课程设置、老师设置、个人页。
- 现场写流：点名提交、学生状态修改、未到学生同步、复制未到文本、系统外老师录入、恢复默认老师、课程学生新增 / 编辑。
- 内测期重点：真实反馈复现、回归验证、后端接口缺口确认、静态导出部署兼容、小步迭代。

## 已知产品约束

- 默认已经接入后端数据，不能用 mock 数据冒充联调完成。
- 业务结论必须区分真实内测环境、本机 shared dev 复现、静态导出验证和代码静态判断。
- 代课、互换、合班、教师端临时学生补录等写链若后端或 shared dev 未验证通过，不能写成完整闭环。
- 老师端和管理端面对同一课节时，点名老师、学生名单、到课状态、未到统计必须保持一致。

## 内测反馈处理口径

记录内测问题时，至少保留：

- 环境：真实内测 / 线上、本机 shared dev、静态产物或代码静态判断。
- 身份：账号角色、校区、老师 / 管理员身份。
- 页面：路径、参数、操作入口。
- 行为：操作步骤、接口请求、页面表现、错误提示。
- 结论：前端问题、后端协同问题、数据问题、部署问题或待复现。
