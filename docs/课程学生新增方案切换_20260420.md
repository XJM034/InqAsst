# 课程学生新增方案切换（2026-04-20）

## 1. 背景

- 用户在管理端学生新增页明确指出两点问题：
  - 页面一直加载
  - 学生 ID 不应由前端生成，而应来源于数据库
- 排查后确认，旧实现存在两个直接问题：
  - 进入新增页后会全量拉取 `/api/admin/students?page=&size=`
  - 未命中已有学生时，前端会生成 `TEMP-*` 临时学生 ID

## 2. 当前正式口径

### 2.1 学生ID

- 管理端页面展示的“学生ID”统一显示数据库 `studentId`
- `externalStudentId` 不再作为管理端展示编号

### 2.2 新增学生路径

分为两条写链：

1. 当前校区已有学生
   - 输入至少 2 个字姓名
   - 调 `GET /api/admin/campuses/{campusId}/students/search?q=&limit=10`
   - 选择已有学生后，保存走 `POST /api/admin/courses/{courseId}/students/existing`

2. 当前校区没有该学生
   - 只填 `name + homeroomClassId`
   - 前端不再生成临时学生 ID
   - 保存前提示“将按临时学生创建，学生ID由数据库生成”
   - 保存走 `POST /api/admin/courses/{courseId}/students`
   - 由后端返回数据库 `studentId`

### 2.3 编辑学生路径

- 继续走 `PUT /api/admin/courses/{courseId}/students/{studentId}`
- 页面只读显示数据库 `studentId`
- `externalStudentId` 不再作为页面必填输入
