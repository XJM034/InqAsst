# 页面对比报告：`/teacher/no-class`

## 1. 先说结论

这页整体上仍然比较接近 mock，但因为教师端底部导航与点名入口链路变了，它需要和 `/teacher/attendance/no-class` 一起重新校验一致性。

## 2. mock 页面想表达什么

mock 的无课页重点很简单：

- 今天没有课
- 这是一个正常状态
- 给老师温和、清楚的反馈

它不是异常页，也不是错误页。

## 3. 当前接口版实际做成了什么

当前实现主要在：

- `app/teacher/no-class/page.tsx`

它仍保留了较多 mock 风格的静态空态特征，但底部 tab 和点名入口的去向已经受当前真实路由影响。

## 4. 页面展示差异

### 4.1 主视觉差异不算大

- 空态结构还在
- 轻量文案还在

### 4.2 链路去向变化更值得关注

因为当前新增了 `/teacher/attendance` 和 `/teacher/attendance/no-class`，所以：

- 老师从不同入口进入“无课”状态时
- 可能落在不同页面

这会造成教师端无课体验被分裂。

## 5. mock 与实际接口数据差异

这页本身偏静态，数据差异不大。

真正的差异在于：

- mock 中无课状态是教师端整体体验的一部分
- 当前接口版里，无课状态被拆散到了多个路由

## 6. 对齐建议

- 统一 `/teacher/no-class` 与 `/teacher/attendance/no-class` 的视觉与文案。
- 明确哪个是主无课页，哪个只是点名入口重定向后的落点。
- 不要让老师在不同入口下看到两套无课页面。

## 7. 对齐时要重点核对的点

- 两个无课页是不是同一套体验。
- 底部导航是否仍符合 mock 的教师端习惯。
- 老师是否能自然返回首页或继续其他操作。
