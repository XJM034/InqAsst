# 页面对比报告：`/admin/course-settings/[courseId]/students`

## 1. 先说结论

这页当前还能看出原型关系，但信息密度已经明显变浅了。它从 mock 里的“今日课程名单页”，退化成了“课程学生列表页”。

问题不在于有没有学生列表，而在于丢掉了“今天这门课为什么在这里、当前点名上下文是什么、哪些学生是临时重点项”这些信息。

## 2. mock 页面想表达什么

mock 里的这页不是通用 roster 页，而是“今天生效课程”的学生名单页。

因此页头需要强调：

- 当前是哪门课
- 上课时间
- 上课地点
- 当前老师
- 当前人数

并且名单里还要支持更强的上下文表达，比如：

- 临时学生高亮
- 搜索行政班
- 从课程配置页顺着上下文进入

## 3. 当前接口版实际做成了什么

当前实现主要在：

- `app/admin/course-settings/[courseId]/students/page.tsx`
- `components/app/admin-course-roster-client.tsx`
- `lib/services/mobile-app.ts` 中的 `getAdminCourseRoster`

当前数据链路是：

- `fetchCourseDetail`
- `fetchAdminCourseStudents`
- `mapAdminCourseRosterData`

这套数据足够画出“某门课的学生列表”，但不足以还原“今日生效课程名单页”的上下文。

## 4. 页面展示差异

### 4.1 页头上下文变少了

mock 页头会给出完整的课程上下文。

当前页头更多只剩：

- 课程标题
- 地点
- 任课老师数量或简化老师信息

弱化了：

- 时间
- 今日生效语义
- 当前点名上下文

### 4.2 搜索语义收窄了

mock 搜索维度通常更像：

- 姓名
- 学生 ID
- 行政班

当前更偏：

- 姓名
- 学号

这使得管理员从“按班找人”的效率下降。

### 4.3 名单页不再像“今日课程名册”

mock 中这页是从“今日生效课程”卡片进入的，所以天然带有“今日正在管这门课”的心理预期。

当前页虽然功能还在，但更像独立的课程通讯录页面。

### 4.4 高亮语义丢了

mock 中的 `highlighted` 类信息可以用来表达：

- 临时加入
- 重点关注
- 需要管理员确认

当前接口版没有自然保留这类状态。

## 5. mock 与实际接口数据差异

### 5.1 mock 数据来源

mock 使用 `adminCourseRosters`。

它能直接服务于 UI 所需的页面语义：

- 课程上下文
- 列表项高亮
- 与“今日课程配置页”的前后衔接

### 5.2 当前接口数据来源

当前使用：

- `getAdminCourseRoster`
- `fetchCourseDetail`
- `fetchAdminCourseStudents`
- `mapAdminCourseRosterData`

它拿到的是：

- 课程详情
- 学生列表

但没拿到：

- 这门课属于今天的原因
- 今日课程规则上下文
- 重点学生或临时名单状态

### 5.3 数据缺口造成的结果

由于缺了“今日生效课程”上下文，页面只能退化成通用 roster。

也就是说：

- 不是 UI 没写好
- 而是上游已经不再把这页当成“当天课程链路的一环”

## 6. 对齐建议

- 恢复页头中的时间、地点、老师、人数、今日状态等完整课程摘要。
- 把从 `/admin/course-settings` 带过来的“今日生效课程”上下文补回来。
- 补 `highlighted` 或等价字段，恢复临时学生/重点学生表达。
- 搜索条件要允许按行政班快速过滤。

## 7. 对齐时要重点核对的点

- 页头是不是一眼能看出“这就是今天那门课”。
- 名单里是否保留了 mock 的重点项表达。
- 从课程设置页点进来后，用户是否会觉得还在同一条链路里，而不是跳进了另一套学生管理页。
