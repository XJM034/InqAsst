# 路由说明：`/teacher/attendance/demo/roster`

## 1. 定位

这个路由属于教师点名链路中的兼容名册入口，不是正式单独设计页面。

## 2. 为什么单独记录

教师端当前已经从 mock 的固定 demo 路线，切到真实 `courseId/courseSessionId` 路线。这个兼容路由如果处理不好，会造成：

- 老链接失效
- 名册页上下文错误
- 教师从旧入口进入时看到错误课程

## 3. 对齐要求

- 保证旧入口仍能稳定落到正确名册场景。
- 不额外扩展新语义，只做兼容。
- 与 `/teacher/home/roster` 的页面语义保持一致。
