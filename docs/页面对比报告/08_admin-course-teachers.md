# 页面对比报告：`/admin/course-teachers`

## 1. 先说结论

这页当前最大的偏差是“还叫课程老师页，但已经看不出老师对应的是哪门课”。页面名称还在，核心关系却被抹平了。

## 2. mock 页面想表达什么

mock 里的课程老师页，本质上是在看“老师和课程的对应关系”。

因此老师卡片上不只是老师信息，还要明确展示：

- 星期维度
- 课程名
- 上课地点
- 上课时间
- 默认老师还是临时代课老师

页面目标是帮助管理员回答：

- 哪个老师教哪门课
- 哪门课当前是谁负责

## 3. 当前接口版实际做成了什么

当前实现主要在：

- `app/admin/course-teachers/page.tsx`
- `components/app/admin-course-teachers-client.tsx`
- `lib/services/mobile-app.ts` 中的 `getAdminCourseTeachersData`

当前数据链路是：

- `fetchAdminTeachers`
- `mapAdminCourseTeachersData`

也就是说，当前页拿到的是“老师列表”，不是“课程老师关系列表”。

## 4. 页面展示差异

### 4.1 页面标题没变，内容语义变了

页面还叫“课程老师”，但卡片内容已经主要是：

- 姓名
- 手机号

课程维度被大幅削弱。

### 4.2 周维度和课程维度消失了

mock 里应该能从页面直接看出：

- 周几
- 哪门课
- 几点上
- 在哪上

当前页更像通讯录。

### 4.3 默认/临时代课状态弱化了

mock 非常强调老师是默认配置还是临时替换。

当前如果没有这层表达，就无法服务“课程老师”这个页面名。

## 5. mock 与实际接口数据差异

### 5.1 mock 数据来源

mock 使用 `adminCourseTeachersData`。

它是为页面设计过的视图模型，天然包含课程维度和老师状态维度。

### 5.2 当前接口数据来源

当前使用：

- `getAdminCourseTeachersData`
- `fetchAdminTeachers`
- `mapAdminCourseTeachersData`

这里最大的根因是接口源头就只给了老师实体列表。

### 5.3 关键缺口

当前接口模型缺少：

- 老师对应课程
- 星期
- 时间
- 地点
- 默认/临时状态

没有这些，页面再怎么调整样式，也还原不了 mock。

## 6. 对齐建议

- 页面如果继续叫“课程老师”，数据源就必须回到“课程-老师关系”而不是纯老师表。
- 至少补出课程摘要、时间地点和默认/临时状态。
- 如果短期内接口给不到，就需要聚合课程和老师配置数据，不能继续直接展示纯教师列表。

## 7. 对齐时要重点核对的点

- 是否一眼能看出“某位老师对应哪门课”。
- 是否能区分默认老师和临时代课老师。
- 是否保留了周几、时间、地点这三个 mock 中的重要场景维度。
