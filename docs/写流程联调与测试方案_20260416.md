# 写流程联调与测试方案（2026-04-16）

## 1. 目标

本轮测试补齐移动端真实写流程验证，并保持以下边界：

- 不修改现有数据库表结构
- 优先通过已有业务接口完成写入与回滚
- 只有在业务接口缺少删除/回滚能力时，才使用测试 helper 清理本次新建的数据
- 清理范围仅限本次测试自己创建的记录

当前工作分支：

- `codex-write-flow-live-test-20260415`

## 2. 本轮覆盖范围

### 2.1 读流程基线

继续保留现有 `smoke`：

- 登录页可访问
- 老师登录、角色校验、今日课表/点名入口
- 管理员登录、角色校验、首页/总控/未到学生/学生表单页只读访问
- 多校区管理员选校区

命令：

```bash
npm run test:e2e:smoke
```

### 2.2 写流程 live

新增 `@write-live` 覆盖两条真实写链路：

1. 管理员学生写流
   - 进入课程学生新增页
   - 新建学生
   - 课程名单页校验
   - 编辑同一学生
   - 接口复核修改结果
   - 使用测试 helper 清理本次新建学生与课程名单关联

2. 管理员课节实际上课时间写流
   - 选择当天可测且 `actualCustomized=false` 的课节
   - 修改实际上课时间
   - 接口复核已生效
   - 调用已有 reset 接口恢复默认
   - 再次复核恢复结果

命令：

```bash
npm run test:e2e:write-live
```

## 3. 数据与环境原则

### 3.1 样本选择原则

- 优先使用当天已有业务数据
- 管理员学生写流优先选择当天 `roll-call overview` 中可见课程；若 overview 为空，则回退到当天 `time-settings sessions`
- 课节时间写流只选择 `actualCustomized=false` 的课节，避免覆盖已有人工配置

### 3.2 数据库触达原则

- 默认不依赖 DB 完成主流程
- DB 只用于：
  - 写流学生数据的兜底清理
  - 必要时核对测试后残留
- 不做 schema / migration 变更
- 不修改现有表结构

## 4. 环境变量

基础账号沿用现有 smoke：

- `E2E_ADMIN_PHONE`
- `E2E_ADMIN_CAMPUS_NAME`
- `E2E_LOGIN_CODE`
- `PLAYWRIGHT_BASE_URL`

学生写流清理额外需要：

- `DAOLEME_DB_URL`
- `DAOLEME_DB_USERNAME`
- `DAOLEME_DB_PASSWORD`
- `DAOLEME_MYSQL_CONNECTOR_JAR`（可选；未配置时按本机 Maven 默认仓库推断）

模板见：

- [`.env.e2e.example`](/d:/proj/backproj/InqAsst/.env.e2e.example:1)

## 5. 执行顺序

1. 启动本地前端与后端联调环境
2. 先跑 `smoke`，确认读链路没有倒退
3. 再跑 `write-live`
4. 若学生写流失败，优先看：
   - `/api/admin/courses/{courseId}/students`
   - `/api/admin/courses/{courseId}/students/{studentId}`
   - 课程学生名单页 `/admin/course-settings/{courseId}/students`
5. 若时间写流失败，优先看：
   - `/api/admin/time-settings/sessions`
   - `/api/admin/course-sessions/{courseSessionId}/time-settings`
   - `/api/admin/course-sessions/{courseSessionId}/actual-time`
   - `/admin/time-settings/{courseSessionId}/picker?kind=actual`

## 6. 清理策略

### 6.1 学生写流

由于当前没有“删除课程学生/删除学生”的现成业务接口，清理策略如下：

- 每次写流使用唯一 `external_student_id`
- 用例结束时只删除满足该唯一标识的本次测试数据
- 删除顺序：
  1. `course_student`
  2. `student`

### 6.2 课节时间写流

不使用 DB 清理，统一走已有业务 reset：

- `DELETE /api/admin/course-sessions/{courseSessionId}/actual-time`

## 7. 通过标准

- `smoke` 全通过
- `@write-live` 全通过
- 学生创建后名单页与接口结果一致
- 学生编辑后接口结果一致
- 课节时间更新后接口结果一致
- reset 后接口恢复默认口径
- 本次测试创建的学生数据无残留

## 8. 已知边界

- 学生写流依赖 DB 兜底清理，因为当前业务接口没有删除能力
- 若当天没有可测课节，时间写流会明确报“无可用样本”，而不是误判前端失败
- 若 DB 环境变量未配置，学生写流会跳过，时间写流仍可单独执行
